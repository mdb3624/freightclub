# 📓 Resilience Logistics Platform: Learnings & Debt Ledger

## 🏗️ Architectural Decision Records (ADR)

### ADR-001: Hexagonal Domain Purity
**Date:** 2026-04-21
**Status:** Accepted
**Decision:** All business logic must reside in the `domain` package using Pure Java (Records, POJOs). No Spring, JPA, or Jakarta dependencies allowed.
**Consequence:** Requires mapping layers (Mappers) to convert Domain Aggregates to Persistence Entities.

### ADR-002: PostgreSQL Native RLS
**Date:** 2026-04-21
**Status:** Accepted
**Decision:** Use Row Level Security (RLS) on the Enon instance via `app.current_tenant` session variables.
**Consequence:** All database adapters must propagate the tenant ID from the Security Context to the DB session.

### ADR-005: Auto-Match Engine (Load → Carrier Matching)
**Date:** 2026-04-21
**Status:** Accepted
**Decision:** `LoadPublishedListener` polls `message_outbox` for `LoadPublishedEvent` entries every 5 seconds. `AutoMatchService` (application layer) uses `CarrierSearchPort` and `MatchRecommendationPort` (output ports) to find eligible carriers and persist `MatchRecommendation` records. Matching logic — `MatchCriteria` and `MatchRecommendation` invariants — lives in the domain layer. Tenant isolation is enforced at the query level (`WHERE tenant_id = :tenantId`) and at the DB level via RLS on `carrier_profiles` and `match_recommendations`.
**Consequence:** `LoadPublishedEvent` now carries `equipmentType` and `originCity`; `LoadAggregate` grows two optional String fields populated on reconstitution from DB.

### ADR-004: Transactional Outbox for Domain Event Delivery
**Date:** 2026-04-21
**Status:** Accepted
**Decision:** Domain events are persisted to a `message_outbox` table within the same database transaction as the aggregate state change. `LoadApplicationService` (the application service) drains pending events from the aggregate, saves the aggregate state, then calls `OutboxDomainEventPublisher.publish()` — all under a single `@Transactional` boundary. A future relay process will poll `status = 'PENDING'` rows and dispatch them.
**Consequence:** Downstream modules (`notification`, `reputation`) will receive events reliably. The outbox table has RLS enabled with `tenant_id` column so the relay inherits tenant isolation. `LoadPublishedEvent`, `LoadClaimedEvent`, and `LoadDeliveredEvent` are now wired into the outbox.

### ADR-003: RS256 JWT with Spring Security OAuth2 Resource Server
**Date:** 2026-04-21
**Status:** Accepted
**Decision:** Replace JJWT HMAC (HS256) with RSA (RS256) for JWT signing/verification. `JwtService` signs with `RSAPrivateKey`; `SecurityConfig` registers a `NimbusJwtDecoder` (via `spring-boot-starter-oauth2-resource-server`) for verification. If no PEM keys are configured, an ephemeral 2048-bit key pair is generated at startup (dev/test only).
**Consequence:** `JwtAuthenticationFilter` is retired from the security chain (auto-registration disabled via `FilterRegistrationBean`). A new `TenantContextFilter` in `com.freightclub.infrastructure.security` runs after `BearerTokenAuthenticationFilter`, reads `tenantId` from the verified `JwtAuthenticationToken`, and binds it to `TenantContextHolder`. Production deployments must supply `JWT_RSA_PRIVATE_KEY` and `JWT_RSA_PUBLIC_KEY` env vars (PEM-encoded PKCS#8 / X.509).
