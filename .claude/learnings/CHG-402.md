# CHG-402: Shipper Reputation Definition Mismatch

**Original Story:** US-402 (Shipper Reputation Profile & Aggregation)  
**Discovered By:** Code Review on 2026-05-27  
**Root Cause:** Story spec and implementation diverged during Phase 4 development

---

## The Mismatch

### Story Spec (AC-402-1, AC-402-2, AC-402-7) Requires:
- Average **star rating** (1-5) from `load_ratings` table
- **Total ratings received** count (denominator)
- **Rating distribution** (breakdown of 1, 2, 3, 4, 5-star counts)

### Actual Implementation Provides:
- Average **payment speed** (days from delivery → payment)
- **Completed load count**
- **Cancelled load count** (while claimed)
- **Open dispute count**

---

## Why This Happened

The spec was written assuming shipper reputation = "how do carriers rate this shipper based on load delivery experience" (bidirectional ratings from US-401).

The implementation evolved into "how reliable is this shipper?" tracking operational/payment metrics instead of carrier-submitted ratings. This is actually MORE useful for trust assessment.

---

## Two Options

### Option A: Keep Implementation As-Is (Recommended)
- Update story spec (US-402.md) to document what's actually implemented
- Payment metrics are MORE predictive of shipper reliability than carrier ratings alone
- Carriers see shipper payment speed, cancel history, dispute patterns
- Requires minimal work: update spec doc, re-test against new AC

**Rationale:** Current implementation is superior to spec. Payment behavior is objective; ratings are subjective. Carriers need both (ratings on load board, payments in profile).

### Option B: Rework Implementation to Match Spec
- Add `avgRating` (avg of rating_value from load_ratings)
- Add `totalRatingsReceived`, `ratingDistribution`
- Remove/deprecate payment metrics
- Lose carrier visibility into shipper payment reliability
- 3–4 hours rework + testing

---

## Recommendation

**Option A:** Align spec to implementation (payment metrics are better).

Then create a separate story (**US-402b: Rating-Based Trust Score**) to display bidirectional ratings on shipper public profile/load board.

This keeps US-402 DONE and unblocks US-403/US-405, while capturing the rating display requirement as future work.

4. Create US-402b ticket if pursuing Option A
