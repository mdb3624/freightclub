# DOC-501 — PWA Document Capture & Offline-First BOL Upload

**As a** Trucker (Owner-Operator),
**I want** to photograph and upload a Bill of Lading (BOL) from my phone — even without a data connection,
**so that** proof of pickup is captured at the dock regardless of signal availability, and synced automatically when connectivity is restored.

## Acceptance Criteria
- [ ] The frontend is registered as a PWA with a service worker that caches the document capture UI shell and queues `POST /api/v1/documents/{loadId}/bol-photo` requests in IndexedDB when offline
- [ ] On reconnection, the service worker background-sync replays queued uploads; a user-visible toast confirms successful sync
- [ ] Backend `DocumentController` accepts `multipart/form-data`; file is validated for MIME type (`image/jpeg`, `image/png`, `application/pdf`) and max size (25 MB) before storage
- [ ] Stored document path is written to `load_documents` with `tenant_id`, `document_type = 'BOL'`, `uploaded_by`, and `created_at`; RLS enforces tenant isolation on reads
- [ ] BOL upload unblocks the `IN_TRANSIT` → `DELIVERED` transition; attempting delivery without a BOL returns `422 Unprocessable Entity`
- [ ] `LocalStorageService` saves files to `${user.home}/freightclub-uploads/{tenant_id}/`; production path is configurable via `app.storage.path`
- [ ] Upload endpoint requires `ROLE_TRUCKER`; shipper read endpoints require `authenticated()`
