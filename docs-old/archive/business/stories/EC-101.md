# EC-101 — Real-Time USDOT Re-Verification at Load Claim

**As the** Platform,
**I want** to re-verify a carrier's USDOT authority status at the exact moment they attempt to claim a load — not just at onboarding,
**so that** a carrier whose authority expired between registration and claim cannot execute a fraudulent pickup.

## Acceptance Criteria
- [ ] `ClaimLoadCommandHandler` (application layer) calls `UsdotVerificationPort.checkCurrentStatus(usdotNumber)` as the first step before the `LoadAggregate.claim()` domain method is invoked; a stale cache entry older than 1 hour is considered invalid and triggers a live FMCSA API call
- [ ] If the FMCSA API returns `authority_status = 'INACTIVE'` or times out AND the cached status is already `INACTIVE`, the claim is rejected with `403 Forbidden` and error code `AUTHORITY_INACTIVE_AT_CLAIM`; a `CarrierAuthorityExpiredEvent` is written to `message_outbox` and to `audit_log`
- [ ] If the FMCSA API times out but the cached status is `ACTIVE` (within 1 hour), the claim is permitted with a `AUTHORITY_UNVERIFIED_TIMEOUT` warning logged — fail-open to avoid disrupting operations
- [ ] `carrier_risk_profiles.last_verified_at TIMESTAMPTZ` is updated on every successful re-verify; composite index on `(tenant_id, usdot_number, last_verified_at)` supports recency lookups — satisfies **NFR-503**
- [ ] Cyclomatic complexity ≤ 10 on `ClaimLoadCommandHandler.execute()` — satisfies **NFR-501**; FMCSA call is delegated to `UsdotVerificationPort`, not inlined
- [ ] JaCoCo branch coverage ≥ 80% for the three paths: active → claim proceeds; inactive → 403; timeout + stale cache → 403 — satisfies **NFR-502**
- [ ] All table writes include `tenant_id`; RLS enforces per-tenant isolation — satisfies **REQ-103**
