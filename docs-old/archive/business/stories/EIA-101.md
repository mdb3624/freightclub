# EIA-101 — EIA Backend Diesel Price Fetch and Cache Service

**As a** Platform,
**I want** a scheduled backend service to fetch, validate, and cache regional diesel prices from the EIA v2 API,
**so that** all FreightClub features have a reliable, up-to-date, single source of truth for diesel pricing without exposing the API key to clients.

## Acceptance Criteria
- [ ] `EiaFuelPriceService` issues a single outbound GET to the EIA v2 endpoint on application startup, retrieving the two most recent weekly observations for all five PADD regions (R1X, R3, R4, R5, R50) — satisfies **FR-01**
- [ ] A `@Scheduled` job refreshes the cache at minimum once every 24 hours without user action — satisfies **FR-02**
- [ ] Cached data is always served; the system never returns blank or zero values when a prior EIA weekly publication exists — satisfies **FR-03**
- [ ] **Security:** `EIA_API_KEY` is read exclusively from server-side environment variable (`${app.eia.api-key}`); it is never forwarded to the frontend, included in any API response, or committed to source control — satisfies **NFR-07**; all EIA calls originate from the backend process — satisfies **NFR-08**
- [ ] Responses are validated against the expected schema (non-null `data` array, `duoarea` matches one of the five known codes, `value` parseable as a positive float) before being written to cache; malformed responses trigger fallback to stale cache and log an error — satisfies **NFR-09**
- [ ] Requests time out after 3,000ms; failed requests use exponential backoff with a maximum of 3 retry attempts before falling back to cached data — satisfies **NFR-01**, **NFR-04**
- [ ] Server-side cache TTL is ≥ 6 hours; no more than 10 EIA API calls are issued per 24-hour window — satisfies **NFR-02**
- [ ] Raw EIA response payloads are not persisted to the database; only the derived `(region, price, delta, period, fetchedAt)` tuple is held in the in-memory cache for the TTL duration — satisfies **NFR-11**
- [ ] Cyclomatic complexity ≤ 10 in `EiaFuelPriceService.fetchAndCache()`; response parsing is delegated to `EiaResponseParser` — satisfies reviewer checklist quality gate
- [ ] JaCoCo branch coverage ≥ 80% including: successful fetch + cache write; timeout → retry exhausted → stale fallback; schema validation failure → fallback; scheduled refresh triggers on interval
