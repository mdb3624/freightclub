# MSG-601 — Per-Load Real-Time Messaging Thread

**As a** Shipper or Trucker,
**I want** to send and receive messages within the context of a specific load,
**so that** all communication is captured in a single, searchable thread tied to the load record — reducing off-platform coordination via phone or email.

## Acceptance Criteria
- [ ] `messages` table: `id UUID`, `tenant_id UUID`, `load_id UUID`, `sender_id UUID`, `body TEXT NOT NULL`, `sent_at TIMESTAMPTZ`, `read_at TIMESTAMPTZ NULL`; RLS on `tenant_id`; composite index on `(tenant_id, load_id, sent_at DESC)` for chronological retrieval — satisfies **REQ-103** and **NFR-503**
- [ ] Messaging is only permitted after load status reaches `CLAIMED`; attempting to send a message on a `DRAFT` or `PUBLISHED` load returns `409 Conflict` with `LOAD_NOT_CLAIMED`
- [ ] `POST /api/v1/messages/{loadId}` creates a message row; `GET /api/v1/messages/{loadId}` returns the thread paginated by `sent_at DESC`; both endpoints require `authenticated()` and matching `tenant_id`
- [ ] Real-time delivery is implemented via Server-Sent Events (SSE): `GET /api/v1/messages/{loadId}/stream`; falls back to 5-second polling if SSE is unsupported by the client
- [ ] Unread count is computed as `SELECT COUNT(*) FROM messages WHERE load_id = ? AND sender_id != :currentUser AND read_at IS NULL`; exposed via `GET /api/v1/messages/unread-count` and surfaced as a badge on the dashboard
- [ ] A `MessageSentEvent` triggers an in-app notification and (if configured) an email to the recipient; dispatched via INF-202 outbox relay with `message_type = 'IN_APP_MESSAGE'`
- [ ] Cyclomatic complexity ≤ 10 in `MessageService.send()` — satisfies **NFR-501**
- [ ] JaCoCo branch coverage ≥ 80% for: message on claimed load → sent; message on unclaimed load → 409; SSE stream connection → events received — satisfies **NFR-502**
