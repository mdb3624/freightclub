# DB-008 — Extract `trucker_cost_profiles` Table

**As an** engineer,
**I want** trucker cost profile data extracted from the `users` table into its own `trucker_cost_profiles` table,
**so that** shipper rows are not polluted with NULL trucker-only columns and future cost history tracking is possible.

## Acceptance Criteria

- [ ] A Flyway migration creates `trucker_cost_profiles` with: `id` (UUID PK), `tenant_id`, `user_id` (FK → users, UNIQUE), `monthly_fixed_costs`, `fuel_price_per_gallon`, `fuel_economy_mpg`, `maintenance_cost_per_mile`, `monthly_miles_target`, `target_margin_percent`, `created_at`, `updated_at`.
- [ ] The migration copies existing cost profile data from `users` rows where role = TRUCKER into the new table.
- [ ] The six cost profile columns are dropped from the `users` table in the same migration (or a follow-up migration, coordinated with the application change).
- [ ] The `User` JPA entity is updated to remove cost profile fields; a new `TruckerCostProfile` entity/repository is created.
- [ ] All existing profitability calculations continue to work correctly after the migration.
- [ ] Shipper rows now contain zero cost profile data (no NULL columns for those fields).
- [ ] Migration follows the Flyway naming convention.
