# BUG-004 — URL Filter Enum Guard: Validate Params Before Cast

**Phase:** 1.2
**Persona:** Trucker
**Status:** ✅ Built

## Story

As a trucker, I want load board URL filter parameters to be validated against known enum values before use so that an invalid or tampered URL cannot cause silent filter failures or runtime errors.

## Background

`TruckerDashboard.tsx` was casting URL params directly: `searchParams.get('equip') as EquipmentType`. Any string — including garbage values from a tampered URL — would be passed into the filter query, producing unexpected results with no error surfaced to the user.

## Acceptance Criteria

- A helper function validates each filter param against its allowed enum values before use
- Invalid values are silently discarded and the filter defaults to "no filter" for that field
- The URL is updated to remove invalid params so sharing a broken URL doesn't lock the user into a broken state
- Existing valid filter URLs continue to work without change
