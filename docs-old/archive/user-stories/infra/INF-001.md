# INF-001 — React `<ErrorBoundary>` in `App.tsx`

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As a user, I want the application to display a meaningful error screen instead of a blank page when a render error occurs so that I know something went wrong and can take action (refresh, navigate away).

## Acceptance Criteria

- `App.tsx` wraps the entire component tree in an `ErrorBoundary` class component
- When any child component throws during render, the boundary catches it and displays a friendly error message
- The error screen includes a "Reload" or "Go to Dashboard" action so the user is not stuck
- The caught error is logged (console or monitoring service) for debugging
- A unit test verifies the boundary renders the fallback when a child throws
