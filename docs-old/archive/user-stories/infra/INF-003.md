# INF-003 — Production Environment Config (`application-prod.yml`)

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As an operator, I want a production Spring profile config with environment variable placeholders for all secrets and environment-specific values so that deployments never require modifying committed files.

## Acceptance Criteria

- `application-prod.yml` exists under `backend/src/main/resources/`
- All secrets (JWT secret, DB password, EIA API key) are referenced as `${ENV_VAR_NAME}` placeholders
- Database URL, port, and connection pool settings are configurable via environment variables
- CORS allowed origins, Actuator exposure, and log level are production-appropriate defaults
- No literal secrets or dev-environment values are present in the file
- The application starts successfully with the `prod` profile when all required env vars are set
