# TST-002 — `RefreshTokenService` Unit Tests

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As a developer, I want `RefreshTokenService` covered by unit tests for rotation and revocation so that the race condition fix (SEC-002) is verifiably correct and protected against future regression.

## Acceptance Criteria

- Rotate: consumes old token, issues new token, marks old as revoked with `revoked_at` timestamp
- Rotate on already-revoked token: throws `TokenRevokedException` (401)
- Rotate on expired token: throws `TokenExpiredException` (401)
- Revoke: marks token as revoked; subsequent rotate throws
- All test cases use `@Mock` — no Spring context loaded
