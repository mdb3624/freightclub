# S-006 — Cancel Load Confirmation Dialog

**As a** shipper,
**I want** a confirmation dialog before canceling a live load,
**so that** I cannot accidentally cancel a load in-flight.

## Acceptance Criteria

- [ ] Clicking "Cancel Load" opens a modal dialog — the cancellation is not executed immediately.
- [ ] The dialog clearly states the load reference (ID or origin→destination) so the shipper confirms they are canceling the correct load.
- [ ] The dialog has two explicit actions: "Yes, cancel this load" (destructive, visually distinct in red) and "Keep load" (safe default).
- [ ] Pressing Escape or clicking outside the dialog closes it without canceling the load.
- [ ] The cancel only executes after the shipper explicitly clicks the confirm button.
- [ ] After successful cancellation, the load status updates to CANCELLED and the dashboard reflects this immediately.
- [ ] If the load is already in a terminal state (DELIVERED, SETTLED, CANCELLED), the cancel action is not available.
