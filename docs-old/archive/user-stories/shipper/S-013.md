# S-013 — Trucker Notification on Load Cancellation

**As a** shipper,
**I want** the trucker notified immediately when I cancel a claimed load,
**so that** they can find a new load without delay.

## Acceptance Criteria

- [ ] When a shipper cancels a load in CLAIMED or IN_TRANSIT status, the assigned trucker receives an in-app notification immediately.
- [ ] The notification includes the load reference (origin → destination) and the cancellation reason.
- [ ] If the trucker has email notifications enabled, an email is also sent within 60 seconds.
- [ ] The trucker's active load slot is freed immediately upon cancellation (they can claim a new load right away).
- [ ] The cancelled load appears in the trucker's load history with status CANCELLED and the reason visible.
- [ ] If the load has no assigned trucker (status OPEN), no trucker notification is sent.
