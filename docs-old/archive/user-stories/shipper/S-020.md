# S-020 — Load Status Change Notifications

**As a** shipper,
**I want** email, SMS, and push notifications on load status changes,
**so that** I don't have to manually refresh the dashboard to track my freight.

## Acceptance Criteria

- [ ] The shipper receives a notification when a load transitions to: CLAIMED, IN_TRANSIT, DELIVERED.
- [ ] Each notification channel (email, SMS, in-app) is sent only if enabled in the shipper's notification preferences (S-003).
- [ ] In-app notifications appear in a notification bell/panel without a page reload.
- [ ] Email notifications are delivered within 60 seconds of the status change.
- [ ] Each notification includes: load reference (origin → destination), new status, and a deep link to the load detail page.
- [ ] Notifications are not sent for DRAFT or CANCELLED transitions unless specifically enabled.
