# S-030 — Email Notification — Load Delivered

**As a** shipper,
**I want** to receive an email when a trucker marks my load as delivered, including the delivery timestamp,
**so that** I have a record of when delivery was confirmed.

## Acceptance Criteria

- [ ] When a load transitions to DELIVERED status, a `LoadEvent` row is created with `event_type = DELIVERED`
- [ ] An email is triggered to the shipper's registered email address
- [ ] The email includes load ID, origin, destination, and delivery timestamp
- [ ] The email includes a reference to any POD (Proof of Delivery) photo if available
- [ ] The email subject clearly identifies the action: "Your load [ID] has been delivered"
- [ ] Email is sent within 5 seconds of the status change
- [ ] If email send fails, the status transition still succeeds and an error is logged
