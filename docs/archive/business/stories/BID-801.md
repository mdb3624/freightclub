# BID-801 — Competitive Load Bidding & Auction Flow

**As a** Shipper,
**I want** to optionally post a load as open-to-bids rather than first-come-first-served,
**so that** I can accept the most competitive rate from a qualified carrier rather than being locked into the first claim.

## Acceptance Criteria
- [ ] `loads` table gains `posting_type VARCHAR DEFAULT 'FCFS' CHECK(posting_type IN ('FCFS', 'BID'))`; `bid_close_at TIMESTAMPTZ NULL` (required when `posting_type = 'BID'`); Flyway migration adds columns; index on `(tenant_id, status, posting_type)` — satisfies **REQ-103** and **NFR-503**
- [ ] `bids` table: `id UUID`, `tenant_id UUID`, `load_id UUID`, `carrier_id UUID`, `proposed_rate NUMERIC(10,2) NOT NULL`, `message TEXT NULL`, `status VARCHAR DEFAULT 'PENDING'`, `submitted_at TIMESTAMPTZ`, `expires_at TIMESTAMPTZ`; unique constraint on `(load_id, carrier_id)`; RLS on `tenant_id`
- [ ] `POST /api/v2/loads/{id}/bids` (requires `ROLE_TRUCKER`) submits a bid; only allowed when `posting_type = 'BID'` and `NOW() < bid_close_at`; attempting to bid on an `FCFS` load returns `409 Conflict` with `LOAD_NOT_BID_TYPE`
- [ ] Shipper reviews bids via `GET /api/v2/loads/{id}/bids`; accepts one via `POST /api/v2/loads/{id}/bids/{bidId}/accept`; acceptance transitions the load `PUBLISHED → CLAIMED` and rejects all other bids atomically (within a single `@Transactional` block)
- [ ] When `NOW() >= bid_close_at` and no bid has been accepted, a `BidWindowExpiredJob` transitions the load back to `PUBLISHED` and notifies the shipper to either extend or re-post
- [ ] Cyclomatic complexity ≤ 10 in `BidService.accept()` — satisfies **NFR-501**; bid rejection is a bulk update, not a loop with individual transactions
- [ ] All bid events (`BidSubmitted`, `BidAccepted`, `BidRejected`, `BidExpired`) are written to `audit_log` — satisfies **REQ-402**
- [ ] JaCoCo branch coverage ≥ 80% for: valid bid submission; bid on FCFS load → 409; accept bid → CLAIMED + others rejected; bid window expired → PUBLISHED — satisfies **NFR-502**
