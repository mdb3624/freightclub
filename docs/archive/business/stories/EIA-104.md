# EIA-104 — Fuel Surcharge Auto-Population in Load Profitability Analyzer

**As a** Driver / Owner-Operator,
**I want** the Fuel Surcharge field in the Load Profitability Analyzer to auto-populate based on current EIA diesel prices and my loaded miles,
**so that** I can instantly see accurate fuel costs without manually looking up prices or doing manual calculations.

## Acceptance Criteria
- [ ] When the user enters loaded miles in the Profitability Analyzer, the Fuel Surcharge field auto-populates with a calculated value: `currentRegionalDieselPrice × miles / MPG_CONSTANT`; the MPG constant is configurable in application properties — satisfies **FR-10**
- [ ] The CPM Calculator reads the current diesel price from the shared `EiaFuelPriceService` cache via an injected dependency; it does not issue a separate fetch to the EIA API — satisfies **FR-11**
- [ ] The auto-populated Fuel Surcharge value is editable; a user-entered override is preserved and not reset on re-render or region change — satisfies **FR-10**
- [ ] If the EIA cache contains no data, the Fuel Surcharge field renders `--` as a placeholder and remains user-editable; the analyzer is not blocked — satisfies **NFR-05** (graceful degradation)
- [ ] **Security / Auth:** The `GET /api/v1/market/fuel-prices` endpoint enforces `authenticated()` and returns 401 to unauthenticated callers; EIA market data is global (not tenant-scoped) but is still gated behind the auth wall — satisfies reviewer security gate
- [ ] Cyclomatic complexity ≤ 10 in `FuelSurchargeCalculator` — satisfies reviewer checklist quality gate
- [ ] JaCoCo branch coverage ≥ 80% including: cache populated → auto-fill; cache empty → placeholder shown; user override → value preserved; miles field cleared → surcharge cleared
