# INF-202 — Transactional Outbox Relay & Event Dispatch Processor

**As a** Platform Administrator,
**I want** a background relay process to continuously poll the `message_outbox` table for pending events and dispatch them to downstream consumers,
**so that** domain events (load state changes, fraud alerts, detention triggers) are delivered at least once — even across application restarts or transient failures.

## Acceptance Criteria
- [ ] `OutboxRelayService` polls `message_outbox WHERE status = 'PENDING' AND tenant_id = app.current_tenant ORDER BY created_at ASC LIMIT 50` every 5 seconds using `@Scheduled`; `set_config` is called before each batch query — satisfies **REQ-202** and **REQ-103**
- [ ] Each row is processed in an individual `@Transactional` block; on success, `status` is updated to `'PROCESSED'` and `processed_at = NOW()`; on failure, `retry_count` is incremented; rows with `retry_count ≥ 5` are set to `'DEAD_LETTER'` and a `DeadLetterAlert` event is logged to `audit_log`
- [ ] `message_outbox` schema: `id UUID`, `tenant_id UUID`, `aggregate_type VARCHAR`, `aggregate_id UUID`, `event_type VARCHAR`, `payload JSONB`, `status VARCHAR`, `retry_count SMALLINT DEFAULT 0`, `created_at TIMESTAMPTZ`, `processed_at TIMESTAMPTZ`; composite index on `(tenant_id, status, created_at)` — satisfies **NFR-503**
- [ ] Idempotency: if the same `id` is processed twice (e.g., crash after update but before commit), the second attempt detects `status != 'PENDING'` and skips silently
- [ ] `OutboxRelayService` cyclomatic complexity ≤ 10 per method — satisfies **NFR-501**; split into `fetchPendingBatch()`, `dispatchEvent()`, and `markProcessed()` if needed
- [ ] Flyway migration creates the `message_outbox` table with `ENABLE ROW LEVEL SECURITY`; the relay service runs under a DB role with `SET ROLE` bypass for its polling query only (cross-tenant batch polling requires superuser or explicit bypass role)
- [ ] Integration test using Testcontainers verifies: event written → relay fires → downstream `ApplicationEventPublisher` receives event within 10 seconds — satisfies **NFR-502**
