# MAT-301 — PostGIS Spatial Carrier Discovery (50-Mile Radius)

**As the** Match Engine,
**I want** to query for carrier profiles whose home region falls within a 50-mile radius of a load's origin point,
**so that** only geographically viable carriers are surfaced for matching, reducing deadhead miles and improving match quality.

## Acceptance Criteria
- [ ] `carrier_profiles` table includes a `home_location GEOGRAPHY(POINT, 4326)` column; Flyway migration creates a GiST spatial index: `CREATE INDEX idx_carrier_home_location ON carrier_profiles USING GIST(home_location)` — satisfies **REQ-301** and **NFR-503**
- [ ] `loads` table includes `origin_location GEOGRAPHY(POINT, 4326)` populated at publish time; Flyway migration creates a GiST index on this column
- [ ] `CarrierSearchPort` (output port) exposes `findCarriersWithinRadius(tenantId, originPoint, radiusMiles)`; the JPA adapter implements it via: `ST_DWithin(c.home_location, :origin, :radiusMetres)` where `radiusMetres = radiusMiles × 1609.34`
- [ ] All spatial queries include `tenant_id = app.current_tenant` filter before the `ST_DWithin` predicate; `set_config` is called at transaction start — satisfies **REQ-103**
- [ ] `EXPLAIN ANALYZE` output for the spatial query must demonstrate an index scan on `idx_carrier_home_location`, not a sequential scan — satisfies **NFR-503**
- [ ] Default radius is 50 miles; overridable per tenant via `tenant_match_config.radius_miles`; radius must be a positive value — domain `RadiusMiles` Value Object rejects `≤ 0`
- [ ] Code must adhere to **NFR-501** (complexity ≤ 10): spatial query construction is in a dedicated `SpatialQueryBuilder` helper
- [ ] Integration test (Testcontainers PostgreSQL 16 + PostGIS) verifies: carrier at 40 miles → included; carrier at 60 miles → excluded; carrier at exactly 50 miles → included (boundary inclusive)
