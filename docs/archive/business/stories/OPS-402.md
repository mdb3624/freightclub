# OPS-402 — Geo-fenced Dwell Time Tracking (PostGIS)

**As a** Shipper,
**I want** to track how long a truck dwells at my facility before departing,
**so that** I can enforce detention billing and flag chronic delay patterns that impact my OTIF (On-Time-In-Full) metrics.

## Acceptance Criteria
- [ ] Facility location is stored as a PostGIS `GEOGRAPHY(POINT, 4326)` column in `facilities`; a 200-metre geo-fence radius is the default and is configurable per facility
- [ ] Trucker's mobile app reports GPS coordinates via `POST /api/v1/tracking/position`; the backend computes `ST_DWithin(position, facility_geom, 200)` to determine if the truck is within the geo-fence
- [ ] `dwell_events` table records `entered_at` and `exited_at` timestamps with `tenant_id` + RLS; dwell duration is computed as `exited_at - entered_at`
- [ ] Dwell exceeding the shipper's configured free-time threshold (default 2 hours) automatically creates a `DetentionAlert` domain event written to `message_outbox`
- [ ] All geo-queries use a spatial index (`CREATE INDEX ... USING GIST`) on the `GEOGRAPHY` column; EXPLAIN output must show index scan, not seq scan, for point-in-polygon queries
- [ ] Dwell time data is scoped to `tenant_id`; PostGIS functions used in queries must not allow cross-tenant geometry leakage
