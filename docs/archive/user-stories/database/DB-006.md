# DB-006 — Load Board Query Indexes

**As an** engineer,
**I want** composite indexes covering the primary load board filter patterns,
**so that** trucker load board queries use index scans instead of full tenant-scoped table scans as data volume grows.

## Acceptance Criteria

- [ ] A Flyway migration adds the following indexes (or confirms they already exist):
  - `idx_loads_tenant_equip_status` on `(tenant_id, equipment_type, status)`
  - `idx_loads_tenant_origin_state` on `(tenant_id, origin_state)`
  - `idx_loads_tenant_pickup_status` on `(tenant_id, pickup_from, status)`
  - `idx_loads_trucker_status` on `(trucker_id, status)`
- [ ] Each index is created `CONCURRENTLY` (or the MySQL equivalent) to avoid locking the table in production.
- [ ] After migration, `EXPLAIN` output for a load board filter query (e.g., filtering by `tenant_id`, `equipment_type`, `status = 'OPEN'`) shows an index seek, not a full scan.
- [ ] Migration follows the Flyway naming convention.
- [ ] No existing unique or primary key indexes are duplicated.
