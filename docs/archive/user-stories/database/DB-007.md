# DB-007 — `loads.payment_terms` NOT NULL

**As an** engineer,
**I want** `loads.payment_terms` to be `NOT NULL`,
**so that** a load cannot be published without payment terms, enforcing business rules at the database level.

## Acceptance Criteria

- [ ] A Flyway migration alters `loads.payment_terms` to `NOT NULL`.
- [ ] Before adding the constraint, the migration backfills any existing NULL rows with a default value (e.g., `'NET_30'`) — the specific default is confirmed with product before the migration is written.
- [ ] After migration, attempting to insert a load without `payment_terms` fails at the database level.
- [ ] The application-layer `LoadForm` already requires `payment_terms`; this story adds the database-level enforcement.
- [ ] Migration follows the Flyway naming convention.
