# BUG-002 — Write `load_events` Row on Every Status Transition

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want every load status transition to write a row to `load_events` so that the event log is populated and Phase 2 timeline and notification features have a reliable data source.

## Background

The `load_events` table migration existed but no status transition in `LoadService` wrote to it. The table was empty in production, making the status timeline feature impossible to build.

## Acceptance Criteria

- `LoadService` writes a `LoadEvent` row for every transition: CREATED, PUBLISHED, CLAIMED, PICKED_UP, DELIVERED, CANCELLED
- Each event records: `load_id`, `tenant_id`, `actor_id`, `event_type`, `created_at`
- Events are immutable — no updates or deletes
- Unit tests verify each transition produces the expected event type
