# BUG-005 — Server-Side Overweight Load Validation

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want the backend to reject loads with weight > 80,000 lbs unless `overweightAcknowledged = true` so that the frontend checkbox is not the only guard against illegal overweight loads being posted.

## Background

The frontend showed a checkbox for overweight acknowledgment but the backend performed no corresponding validation. A direct API call bypassing the UI could create an overweight load without acknowledgment.

## Acceptance Criteria

- `LoadService` validates: if `weightLbs > 80000` and `overweightAcknowledged != true`, throw a 400 Bad Request
- Validation applies on both create and update
- The error message clearly states the overweight acknowledgment requirement
- Unit test covers: valid weight passes, overweight + acknowledged passes, overweight + not acknowledged fails
