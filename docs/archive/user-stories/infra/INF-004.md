# INF-004 — Structured Logging with Correlation IDs

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As an operator, I want every request to be assigned a correlation ID that appears in all log lines for that request so that I can trace a specific user action across the full log stream when debugging production issues.

## Acceptance Criteria

- A servlet filter generates a UUID correlation ID for each incoming request
- The correlation ID is stored in MDC (`X-Correlation-ID`) for the duration of the request
- All log output includes the correlation ID field
- The correlation ID is returned in the response as `X-Correlation-ID` header so clients can reference it when reporting issues
- Log format is structured (JSON or key=value) to support log aggregation tools
