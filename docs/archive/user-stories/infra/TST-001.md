# TST-001 — `AuthService` Unit Tests

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As a developer, I want `AuthService` covered by unit tests for register, login, refresh, and logout so that the security-critical authentication paths have a safety net against regressions.

## Acceptance Criteria

- Register: valid input creates user + returns tokens; duplicate email returns 409; weak password rejected
- Login: correct credentials return access + refresh tokens; wrong password returns 401; unknown email returns 401
- Refresh: valid refresh cookie rotates tokens; expired token returns 401; revoked token returns 401
- Logout: invalidates the refresh token in DB; subsequent refresh with same token returns 401
- All test cases use `@Mock` dependencies — no Spring context loaded
