# P-004 — SMS Notifications (opt-in, Twilio)

**As a** user (shipper or trucker),
**I want** to opt in to SMS notifications for load status changes,
**so that** I receive alerts on my phone even when I don't have the app open.

## Acceptance Criteria

- [ ] In the user profile settings, a "Notification preferences" section includes an SMS opt-in checkbox
- [ ] The checkbox is labeled "Receive SMS alerts for load status changes"
- [ ] When enabling SMS, the user is prompted to enter their phone number and verify it via SMS code
- [ ] After verification, SMS notifications are sent for: load claimed, load picked up, load delivered, load cancelled
- [ ] SMS messages include the load ID, action, and a link to open the app (or web dashboard)
- [ ] Messages are sent via Twilio (or equivalent SMS provider)
- [ ] SMS send failures do not block the status transition (send async, log errors)
- [ ] Users can disable SMS notifications at any time from their profile settings
- [ ] Phone numbers are stored securely (encrypted at rest)
