# P-005 — EIA Diesel Price Backend Proxy + Cache

**As the** platform,
**I want** a backend endpoint that proxies the EIA Diesel Price API with server-side caching,
**so that** the API key is never exposed to the browser and repeated requests don't exhaust the quota.

## Acceptance Criteria

- [ ] A new endpoint `GET /api/v1/market/diesel-prices` is implemented on the backend
- [ ] The endpoint proxies calls to the U.S. EIA Open Data API v2 (https://api.eia.gov/...)
- [ ] The EIA API key is stored in an environment variable (never committed to source control)
- [ ] The endpoint extracts and returns two prices: DIESEL WEST (R50 duoarea) and DIESEL SOUTH (R4 duoarea)
- [ ] Responses are cached in-memory or via Redis with a 6-hour TTL
- [ ] Cache key includes the date so that stale data can be detected (e.g. cache from 2 days ago)
- [ ] If the cache is stale (> 48 hours old) but EIA is unreachable, the endpoint returns the stale cache with a stale indicator
- [ ] If both cache and EIA are unavailable, the endpoint returns HTTP 503 with a clear error message
- [ ] The response includes a `last_updated` timestamp so the frontend can display staleness
- [ ] The endpoint is unauthenticated (no JWT required) to allow public access
- [ ] Rate limiting is applied (e.g. 10 requests/minute per IP) to prevent abuse
