# SEC-003 — Rate Limiting on Auth Endpoints

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want login and registration endpoints throttled by a token-bucket rate limiter so that brute force and credential stuffing attacks are blocked before they can exhaust valid credentials.

## Acceptance Criteria

- `POST /api/v1/auth/login` and `POST /api/v1/auth/register` are rate-limited to 10 requests per IP per minute
- Requests exceeding the limit receive `429 Too Many Requests` with a `Retry-After` header
- The limit resets after 60 seconds
- Legitimate users on shared IPs are not blocked under normal usage patterns
- Implementation uses Bucket4j (`AuthRateLimitFilter`)
