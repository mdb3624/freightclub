# SEC-004 — JWT `iss` and `aud` Claims

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want every JWT to include issuer (`iss`) and audience (`aud`) claims, and for the filter to validate both on every request, so that tokens are bound to FreightClub and cannot be replayed against other services.

## Acceptance Criteria

- All issued JWTs include `iss: freightclub` and `aud: freightclub-api`
- `JwtService` validates `iss` and `aud` on every token parse; tokens missing or mismatching either claim are rejected with 401
- Unit tests cover: valid token accepted, wrong issuer rejected, wrong audience rejected, missing claim rejected

## Notes

See ADR-009 in ARCHITECTURE.md. Tokens already in circulation at deployment time will fail validation and force a re-login — acceptable for a pre-production system.
