# SEC-005 — JWT Secret to Environment Variable

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want the JWT signing secret stored in an environment variable rather than committed in `application-dev.yml` so that the secret cannot be extracted from the source repository.

## Acceptance Criteria

- The hex JWT secret is removed from `application-dev.yml` and replaced with a `${JWT_SECRET}` placeholder
- `application-prod.yml` references the same placeholder
- A `.env.example` or README note documents the required environment variable
- The application fails to start with a clear error if `JWT_SECRET` is not set
- The old secret value is not present in any committed file
