# SEC-006 — Remove Hardcoded Tailscale Domain from `vite.config.ts`

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want developer-specific hostnames removed from `vite.config.ts` and replaced with an environment variable so that personal infrastructure is never committed to the shared repository.

## Acceptance Criteria

- Any hardcoded Tailscale or developer-specific hostname is removed from `vite.config.ts`
- External hostnames (for Tailscale or other remote access) are read from `VITE_ALLOWED_HOSTS` environment variable
- The default config (no env var set) works correctly for local development on `localhost`
- `vite.config.ts` is committed with no personal hostnames present
