# T-009 — Filter State Persisted in URL

**As a** trucker,
**I want** my load board filter selections persisted in the URL,
**so that** navigating back from a load detail page doesn't reset all my filters and force repetitive re-entry.

## Acceptance Criteria

- [ ] Active filters (origin state, destination state, equipment type, pickup date, sort column, sort direction) are reflected as URL query parameters (e.g., `?origin=IL&dest=OH&equip=DRY_VAN`).
- [ ] URL updates in real time as the trucker changes filters, without a page reload.
- [ ] Navigating back from a load detail page restores the exact filter state (all params preserved).
- [ ] A filtered URL can be bookmarked and shared; opening it applies the same filters.
- [ ] Invalid or unrecognized query params are ignored gracefully (no error, filters default to "all").
- [ ] Direct navigation to `/loads` (no params) shows the unfiltered load board.
