# T-022 — Payment Notification After Delivery

**As a** trucker,
**I want** to receive a payment notification after delivery confirmation,
**so that** I know when to expect funds and can follow up if payment is delayed.

## Acceptance Criteria

- [ ] When a load transitions to DELIVERED, the trucker receives an in-app notification: "Your delivery has been confirmed. Payment of $X is due within [payment terms] days."
- [ ] If the shipper initiates payment in-platform, the trucker receives a second notification: "Payment of $X has been sent. Expected arrival: [date]."
- [ ] If payment has not been initiated within the agreed payment terms window, the trucker receives a reminder notification.
- [ ] Notifications include the load reference and a deep link to the load detail/payment page.
- [ ] Notifications respect the trucker's notification channel preferences (in-app, email, SMS).
