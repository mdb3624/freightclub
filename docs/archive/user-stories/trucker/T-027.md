# T-027 — EIA Fuel Surcharge Auto-Populate in Profitability Analyzer

**As a** trucker,
**I want** the Load Profitability Analyzer to auto-populate a fuel surcharge from the current diesel price and estimated miles,
**so that** my profitability calculation uses real-world fuel costs without manual entry.

## Acceptance Criteria

- [ ] When viewing a load detail page, the profitability analyzer includes a "Fuel surcharge" input field
- [ ] The field is pre-populated with a value calculated as: `current_diesel_price × estimated_miles`
- [ ] The estimated miles are calculated from the load's origin and destination (or derived from load distance if available)
- [ ] The current diesel price is fetched from the shared EIA price store (P-005) — same source as the market ticker
- [ ] The trucker can override the auto-populated value by editing the field
- [ ] If the EIA price is unavailable (API outage, cache expired), the field shows a placeholder and allows manual entry
- [ ] The fuel surcharge is included in the final profitability calculation: `total_profit = revenue - fuel_cost - maintenance_cost - fuel_surcharge`
- [ ] Changes to the fuel surcharge input immediately update the profitability breakdown and RPM calculation
