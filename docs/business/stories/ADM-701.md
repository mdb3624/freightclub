# ADM-701 — System-wide Fraud Monitoring & USDOT Risk Flagging

**As a** Platform Administrator,
**I want** the platform to automatically flag carriers with invalid or high-risk USDOT numbers,
**so that** shippers are protected from double-brokering, cargo theft, and uninsured carrier fraud before a load is claimed.

## Acceptance Criteria
- [ ] `CarrierRiskProfile` domain entity stores `usdotNumber`, `riskScore` (0–100), `insuranceExpiryDate`, `flaggedReasons` (JSONB array), and `lastVerifiedAt TIMESTAMPTZ`
- [ ] On carrier registration, `UsdotVerificationService` (application layer) calls the FMCSA SAFER Web API; a failed or timed-out call sets `riskScore = -1` (unverified) and schedules a retry via `message_outbox`
- [ ] Automated risk rules (cyclomatic complexity ≤ 10 per rule method): expired insurance → score penalty 40; out-of-service order active → score penalty 60; authority revoked → score penalty 100 (auto-block)
- [ ] `CarrierRiskProfile` with `riskScore ≥ 70` triggers a `FraudAlertEvent` written to `message_outbox`; an in-app notification is sent to all SHIPPER users in the affected tenant
- [ ] Load claiming (`OPS-401`) is blocked for any carrier with `riskScore = 100` (revoked authority); returns `403 Forbidden` with error code `CARRIER_AUTHORITY_REVOKED`
- [ ] `carrier_risk_profiles` table has `tenant_id` + RLS; cross-tenant risk data is never exposed — each tenant manages their approved carrier roster independently
- [ ] USDOT verification results are cached for 24 hours in the `carrier_risk_profiles` table; a background job refreshes records where `lastVerifiedAt < NOW() - INTERVAL '24 hours'`
- [ ] All fraud flag events are written to `audit_log` (AUD-601) with `event_type = 'CARRIER_RISK_FLAGGED'`

## Related Stories

- OPS-401 (risk check gates load claiming)
- AUD-601 (fraud flags are immutably logged)
