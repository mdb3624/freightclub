# ADM-702 — Double Brokering Detection via USDOT Pattern Analysis

**As a** Platform Administrator,
**I want** the system to detect and flag tenants whose USDOT history exhibits patterns associated with double brokering,
**so that** shippers are protected from cargo theft and uninsured liability exposure caused by loads being re-brokered to unvetted third parties.

## Acceptance Criteria
- [ ] `DoubleBrokeringRiskService` (application layer) evaluates a carrier's USDOT history from `carrier_risk_profiles`; flags are raised when **any** of these patterns are detected — satisfies **REQ-403**:
  - Authority reinstated ≥ 3 times within 24 months
  - Operating under a name that differs from the FMCSA-registered DBA by > 30% Levenshtein distance
  - Insurance carrier changed ≥ 2 times within 12 months
- [ ] Each detection rule is implemented as a separate `DoubleBrokeringRule` strategy implementing a `RiskRule` interface; `DoubleBrokeringRiskService.evaluate()` iterates the rule list and aggregates scores — strategy pattern satisfies the Reviewer Checklist's architecture gate
- [ ] Each `RiskRule.evaluate(CarrierRiskProfile)` method has cyclomatic complexity ≤ 10 — satisfies **NFR-501**
- [ ] Detected risk is recorded as a `DOUBLE_BROKERING_SUSPECTED` entry in `audit_log` with full USDOT snapshot payload — satisfies **REQ-402**
- [ ] Carrier with an active double-brokering flag is blocked from claiming loads; `OPS-401` checks `carrierRiskProfileRepository.hasSuspectedDoubleBrokering(tenantId, carrierId)` before allowing the `CLAIMED` transition; returns `403 Forbidden` with code `DOUBLE_BROKERING_SUSPECTED`
- [ ] `carrier_risk_profiles.double_brokering_flag BOOLEAN DEFAULT FALSE` column; index on `(tenant_id, double_brokering_flag)` for fast gate checks — satisfies **NFR-503**
- [ ] Shipper receives an in-app notification when a carrier on their active load is flagged post-claim; notification dispatched via `message_outbox`
- [ ] `DoubleBrokeringRiskServiceTest` covers all three rule triggers and the clean-carrier path; JaCoCo branch coverage ≥ 80% — satisfies **NFR-502**
