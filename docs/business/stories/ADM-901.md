# ADM-901 — Admin Operations Dashboard, Dispute Resolution & Platform Health Metrics

**As a** Platform Administrator,
**I want** a centralised operations dashboard to view users, loads, disputes, and platform health metrics,
**so that** I can identify operational bottlenecks, resolve disputes, and act on fraud signals without direct database access.

## Acceptance Criteria
- [ ] `GET /api/v1/admin/dashboard` (requires `ROLE_ADMIN`) returns a summary payload: `{ activeTenants, publishedLoads, claimedLoads, openDisputes, avgTimeToClaimMinutes, fraudFlagsLast30Days }`; all aggregate queries include `ROLE_ADMIN` bypass of per-tenant RLS (via a dedicated `admin_bypass` DB role)
- [ ] Admin load search `GET /api/v1/admin/loads?status=&tenantId=&carrierId=&dateFrom=&dateTo=` returns paginated results; underlying query uses a partial index on `(tenant_id, status)` with `LIMIT/OFFSET`; page size capped at 100 — satisfies **NFR-503**
- [ ] Admin dispute queue `GET /api/v1/admin/disputes?status=OPEN` returns open disputes ordered by `opened_at ASC`; `PATCH /api/v1/admin/disputes/{id}` resolves them (FIN-504); access restricted to `ROLE_ADMIN` — satisfies **REQ-103** (admin bypass is explicit, not an RLS hole)
- [ ] Admin user management: `GET /api/v1/admin/users?tenantId=` lists users; `PATCH /api/v1/admin/users/{id}/deactivate` soft-deletes (`deleted_at = NOW()`) and immediately invalidates all active refresh tokens for the user; `DeactivationEvent` written to `audit_log` — satisfies **REQ-402**
- [ ] Rate benchmarking: `GET /api/v1/admin/market/benchmark?origin=&destination=&equipment=` returns the median rate for the lane computed from `loads WHERE status = 'DELIVERED' AND created_at > NOW() - INTERVAL '90 days'`; result is cached for 1 hour in an application-level cache
- [ ] Cyclomatic complexity ≤ 10 in all admin query handlers — satisfies **NFR-501**; dashboard aggregation is performed in a single SQL query with CTEs, not via N+1 application loops
- [ ] JaCoCo branch coverage ≥ 80% for: dashboard summary query; dispute resolution; user deactivation + token revocation — satisfies **NFR-502**
