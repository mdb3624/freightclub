# CAR-701 — Carrier Fleet Profile, Preferred Lanes & Shipper Preferred Network

**As a** Trucker,
**I want** to maintain a fleet profile with equipment details and preferred lanes,
**so that** the match engine can surface relevant loads to me and shippers can identify me as a trusted carrier for direct assignment.

## Acceptance Criteria
- [ ] `carrier_profiles` table (already partially defined by MAT-301) gains columns: `trailer_length_ft SMALLINT`, `max_payload_lbs INT`, `preferred_lane_origins JSONB`, `preferred_lane_destinations JSONB`, `availability_days SMALLINT[]`, `available_from TIME`, `available_to TIME`; Flyway migration adds columns with `NULL` defaults; RLS on `tenant_id` — satisfies **REQ-103**
- [ ] `shipper_preferred_carriers` table: `id UUID`, `tenant_id UUID`, `shipper_id UUID`, `carrier_id UUID`, `added_at TIMESTAMPTZ`, `blocked BOOLEAN DEFAULT FALSE`; unique constraint on `(tenant_id, shipper_id, carrier_id)`; index on `(tenant_id, shipper_id, blocked)` — satisfies **NFR-503**
- [ ] `PUT /api/v1/carriers/profile` (requires `ROLE_TRUCKER`) updates the carrier's profile; `GET /api/v1/carriers/{carrierId}/public` returns a sanitised public view (rating, equipment, preferred lanes) accessible to authenticated users with matching `tenant_id`
- [ ] Direct assignment: `POST /api/v2/loads/{id}/assign/{carrierId}` (requires `ROLE_SHIPPER`) bypasses the open board and transitions status `PUBLISHED → CLAIMED`; only permitted if `carrier_id` is in `shipper_preferred_carriers` and `blocked = FALSE`
- [ ] A blocked carrier (`blocked = TRUE`) cannot claim any load from that shipper; `OPS-401.claim()` checks `shipper_preferred_carriers` for block status before allowing the transition; returns `403 Forbidden` with `CARRIER_BLOCKED_BY_SHIPPER`
- [ ] `SuggestedLoadsService` uses `carrier_profiles.preferred_lane_origins/destinations` to filter the load board for the trucker's saved preferences; complexity ≤ 10 — satisfies **NFR-501**
- [ ] JaCoCo branch coverage ≥ 80% for: preferred carrier direct assign → CLAIMED; blocked carrier claim attempt → 403; public profile retrieval → sanitised view — satisfies **NFR-502**
