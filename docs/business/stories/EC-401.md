# EC-401 — BOL Document Validation & Human-in-the-Loop Pending Verification

**As the** Platform,
**I want** to detect likely-invalid BOL uploads (blurry, wrong document type, illegible) and route them to a `PENDING_VERIFICATION` state rather than auto-advancing the load,
**so that** shippers are never deceived by fraudulent or accidental document submissions, and the delivery confirmation chain remains trustworthy.

## Acceptance Criteria
- [ ] `DocumentValidationService` (application layer) performs heuristic checks after upload: file size < 10 KB → likely empty/corrupt; aspect ratio outside 0.5–3.0 → likely not a document page; MIME type mismatch (e.g., `image/gif`) → rejected outright; each rule is a separate `ValidationRule` strategy — satisfies **NFR-501** (complexity ≤ 10 per rule)
- [ ] If any validation rule returns `FAIL`, the `load_documents` row is written with `verification_status = 'PENDING_VERIFICATION'` (not `APPROVED`); the load does NOT advance to the next state; a `DocumentPendingVerificationEvent` is written to `message_outbox` and notifies the shipper
- [ ] Shipper can manually review the document via `GET /api/v1/documents/{loadId}/bol-photo` and either approve (`PATCH ...?action=approve`) or reject (`PATCH ...?action=reject`); approval sets `verification_status = 'APPROVED'` and allows the state machine to proceed
- [ ] `load_documents.verification_status` column: `VARCHAR DEFAULT 'PENDING_VERIFICATION'`; Flyway migration adds column with a CHECK constraint limiting values to `('PENDING_VERIFICATION', 'APPROVED', 'REJECTED')`; index on `(tenant_id, load_id, verification_status)` — satisfies **REQ-103** and **NFR-503**
- [ ] `IN_TRANSIT → ARRIVED → DELIVERED` transitions check `verification_status = 'APPROVED'` on all required documents; `PENDING_VERIFICATION` blocks the transition with `422 Unprocessable Entity` and error code `DOCUMENT_PENDING_VERIFICATION`
- [ ] All verification status changes are recorded in `audit_log` with `event_type = 'DOCUMENT_VERIFICATION_STATUS_CHANGED'` — satisfies **REQ-402**
- [ ] JaCoCo branch coverage ≥ 80% for all validation paths and both approve/reject branches — satisfies **NFR-502**
