# EC-403 — Negative Margin Override Gate & Admin Approval Flow

**As a** Shipper,
**I want** the platform to block any rate adjustment (lumper fees, accessorials, or load edits) that would result in a net-negative tenant margin without explicit Admin approval,
**so that** financial losses are never silently absorbed by the platform and every margin-negative decision is a conscious, audited override.

## Acceptance Criteria
- [ ] `MarginCalculationService` (application layer) computes `netMargin = loadRate - (totalMiles * carrierCPM) - detentionCharges - lumperFees - platformFee`; a `NetMargin` Value Object (domain layer, no Spring/JPA annotations) enforces the margin value type with no constraint on sign — it does NOT throw; it simply flags negative values
- [ ] When any rate adjustment endpoint (`PATCH /api/v2/loads/{id}/rate`) would produce `NetMargin.isNegative() == true`, the request is rejected with `409 Conflict` and error code `NEGATIVE_MARGIN_REQUIRES_OVERRIDE`; the response body includes the computed margin value
- [ ] An Admin can approve the adjustment via `POST /api/v1/admin/loads/{id}/margin-override` with a mandatory `justification` field; the override creates a `MarginOverrideApprovedEvent` in `message_outbox` and `audit_log` — satisfies **REQ-402**
- [ ] `margin_overrides` table: `id UUID`, `tenant_id UUID`, `load_id UUID`, `approved_by UUID`, `justification TEXT NOT NULL`, `pre_override_margin NUMERIC(10,4)`, `post_override_margin NUMERIC(10,4)`, `approved_at TIMESTAMPTZ`; RLS on `tenant_id` — satisfies **REQ-103**; composite index on `(tenant_id, load_id)` — satisfies **NFR-503**
- [ ] Cyclomatic complexity ≤ 10 in `MarginCalculationService.compute()` — satisfies **NFR-501**; individual fee components are resolved via separate `FeeResolver` strategies
- [ ] Admin override endpoint is restricted to `ROLE_ADMIN`; auditor can view all overrides via `GET /api/v1/admin/margin-overrides?loadId={id}`
- [ ] JaCoCo branch coverage ≥ 80% for: positive margin → allowed; negative margin → blocked; admin override → applied — satisfies **NFR-502**
