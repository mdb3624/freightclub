# MAT-302 — Strategic Profitability Scoring (ELROD Calculation)

**As a** Trucker (Owner-Operator),
**I want** each available load to display a profitability score based on my operating costs,
**so that** I can prioritize high-value hauls and avoid loads that erode margin after fuel, tolls, and dwell time.

## Acceptance Criteria
- [ ] `MatchRecommendation` domain object stores a `profitabilityScore` computed by the ELROD formula: `(rate − (distanceMiles × costPerMile)) / distanceMiles`; negative scores are valid and must not be filtered out by the engine
- [ ] `MatchCriteria` encapsulates carrier equipment type, home region, and minimum acceptable RPM (revenue-per-mile) threshold as a domain Value Object — no Spring annotations in the `domain/` package
- [ ] `AutoMatchService` (application layer) queries `CarrierSearchPort` with equipment-type and origin proximity filters before scoring; cyclomatic complexity of the scoring method must not exceed 10
- [ ] Current diesel price from the EIA cache is injected into the ELROD cost-per-mile calculation; if EIA is unavailable, a configurable fallback rate (`app.eia.fallback-cpp`) is used
- [ ] Score badges rendered in the UI use the established RPM colour scheme: Green (score ≥ carrier's RPM threshold), Yellow (0–threshold), Red (negative)
- [ ] `MatchRecommendation` records are tenant-isolated via `tenant_id` column and RLS policy; a carrier can never read recommendations seeded for a different tenant
