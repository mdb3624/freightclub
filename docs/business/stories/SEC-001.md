# SEC-001: Add @PreAuthorize Annotations to DELETE/PUT Endpoints

**Status:** 🔴 CRITICAL — Security Vulnerability  
**Priority:** P1 (Blocking)  
**Assigned To:** Coder + Reviewer  
**Estimated Effort:** 3 hours  
**Created:** 2026-05-22

---

## Problem Statement

Currently, **0 endpoints in the 8 backend controllers have @PreAuthorize annotations**. Any authenticated user can delete or modify loads, profiles, and documents from other tenants.

**Impact:** CRITICAL security vulnerability. Multi-tenancy enforcement relies solely on application-layer checks; no endpoint-level authorization gates exist.

---

## Acceptance Criteria

**AC-001-1:** All DELETE endpoints protected with @PreAuthorize  
- [ ] LoadController.deleteLoad() → `@PreAuthorize("@loadService.isOwner(#id)")`
- [ ] ProfileController.deleteProfile() → `@PreAuthorize("@profileService.isOwner(#id)")`
- [ ] DocumentController.deleteDocument() → `@PreAuthorize("@documentService.isOwner(#id)")`

**AC-001-2:** All PUT/PATCH mutation endpoints protected with @PreAuthorize  
- [ ] LoadController.publishLoad() → `@PreAuthorize("@loadService.isOwner(#id)")`
- [ ] LoadController.updateLoad() → `@PreAuthorize("@loadService.isOwner(#id)")`
- [ ] ProfileController.updateProfile() → `@PreAuthorize("@profileService.isOwner(#id)")`
- [ ] DocumentController.updateDocument() → `@PreAuthorize("@documentService.isOwner(#id)")`

**AC-001-3:** Authorization check methods implemented in service layer  
- [ ] LoadService.isOwner(loadId) → verify load.tenantId == TenantContextHolder.getTenantId()
- [ ] ProfileService.isOwner(profileId) → verify profile.tenantId == TenantContextHolder.getTenantId()
- [ ] DocumentService.isOwner(documentId) → verify document.tenantId == TenantContextHolder.getTenantId()

**AC-001-4:** Unit tests verify authorization gates  
- [ ] Test: Unauthorized user cannot DELETE other tenant's load → 403 Forbidden
- [ ] Test: Unauthorized user cannot PUT other tenant's profile → 403 Forbidden
- [ ] Test: Authorized owner can DELETE/PUT own resources → 200 OK
- [ ] Test coverage: ≥80% branch coverage on authorization logic

**AC-001-5:** Code review gate passed  
- [ ] Reviewer confirms all mutation endpoints protected
- [ ] Reviewer confirms isOwner() methods verify tenant_id via TenantContextHolder
- [ ] No hardcoded role checks (e.g., "ADMIN"); use data-driven ownership

---

## Implementation Notes

- Use `@PreAuthorize("@serviceName.isOwner(#id)")` to call service-layer methods
- Service methods must check: `resource.tenantId == TenantContextHolder.getTenantId()`
- Return 403 Forbidden (not 404) if user lacks authorization — prevents information leakage
- Add @EnableGlobalMethodSecurity to SecurityConfig if not present

## Testing

```java
@Test
void testUnauthorizedUserCannotDeleteOtherTenantLoad() {
  // Setup: Load owned by Tenant A
  Load load = new Load(id="load-1", tenantId="tenant-a");
  
  // Act: User from Tenant B tries to delete
  TenantContextHolder.setTenantId("tenant-b");
  
  // Assert: 403 Forbidden
  assertThrows(AccessDeniedException.class, () -> loadController.deleteLoad("load-1"));
}
```
