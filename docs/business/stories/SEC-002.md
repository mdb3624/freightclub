## Problem Statement

Five core tables created in 2026 migrations lack PostgreSQL Row-Level Security (RLS) policies. Multi-tenancy enforcement relies entirely on application-layer checks via TenantContextHolder; the database itself provides **no enforcement**.

**Risk:** If app-layer filtering is bypassed (SQL injection, misconfiguration), users can access other tenants' data at the database level.

**Tables Affected:**
1. message_outbox — Phase 6 (In-App Messaging)
2. shipper_profiles — Phase 7 (Carrier Management)
3. payment_accounts — Phase 5 (Payments & Settlement)
4. load_recommendations — Phase 7 (Carrier Management)
5. carrier_cost_profiles — Phase 7 (Carrier Management)

```sql
CREATE POLICY tenant_isolation ON <table_name> 
FOR SELECT

```

```sql

USING (tenant_id = CURRENT_SETTING('app.current_tenant_id')::VARCHAR(36));
```

## Implementation Notes

- **TenantContextHolder Integration:** The JPA/Hibernate interceptor sets `app.current_tenant_id` via `set_config()` on each request
- **Testing:** Use two separate database sessions with different tenant IDs to verify RLS blocks cross-tenant access
- **Flyway:** Create a single migration file for all 5 tables; use conditional CREATE POLICY statements to handle idempotency

## Testing

```java
@Test
void testRLSBlocksCrossTenantSelectMessage() {
  // Setup: Message owned by Tenant A
  Message msg = messageRepo.save(new Message(id="msg-1", tenantId="tenant-a"));
  
  // Act: Query as Tenant B (RLS policy active)
  TenantContextHolder.setTenantId("tenant-b");
  List<Message> results = messageRepo.findAll(); // RLS filters in database
  
  // Assert: Tenant B cannot see Tenant A's message
  assertTrue(results.isEmpty(), "RLS should block cross-tenant SELECT");
}

@Test
void testRLSBlocksCrossTenantUpdateProfile() {
  // Setup: Profile owned by Tenant A
  Profile profile = profileRepo.save(new Profile(id="prof-1", tenantId="tenant-a"));
  
  // Act: Try to UPDATE as Tenant B
  TenantContextHolder.setTenantId("tenant-b");
  profile.setName("Hacked");
  
  // Assert: RLS blocks UPDATE
  assertThrows(DataAccessException.class, () -> profileRepo.save(profile));
}
```
