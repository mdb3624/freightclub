# SYS-102 — Tenant Invitation & RBAC

**As a** Platform Administrator,
**I want** to invite users to a tenant with a pre-assigned role (SHIPPER or TRUCKER),
**so that** access to the platform is scoped, auditable, and enforced at the database layer without relying on application-level checks alone.

## Acceptance Criteria
- [ ] Invitation is created with a one-time join code tied to a `tenant_id` and an expiry timestamp; stored in `tenant_invitations` with `used_at IS NULL` guard
- [ ] Accepting an invitation creates a `users` row with the correct `tenant_id` and `role`; the join code is immediately marked `used_at = NOW()`
- [ ] PostgreSQL RLS policy on `users` enforces `tenant_id = app.current_tenant` — no cross-tenant user enumeration is possible
- [ ] Attempting to reuse an expired or consumed join code returns `409 Conflict`
- [ ] Role assignment is immutable post-registration; a separate admin action is required to change roles (audit logged)
- [ ] `UserDetailsServiceImpl` loads authorities from the `role` column; Spring Security `hasRole()` checks pass without custom filters

## Related Stories
- LOAD-201 (tenant isolation flows from the role established here)
- AUD-601 (role change events must appear in the immutable audit trail)
