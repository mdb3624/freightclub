# US-101: Multi-Tenant Registration

**Phase:** 1 — Core Load Lifecycle  
**Status:** COMPLETED  
**Depends On:** None  
**Blocks:** US-102, US-103, US-104

---

## User Story

As a **new user (shipper or trucker)**, I need to register with an email and password so that I can create an account in the FreightClub platform and access its features within my tenant organization.

---

## Acceptance Criteria

| ID | Criterion | Status |
|---|---|---|
| AC1 | A user can register with a valid email and password via the RegisterForm | ✅ |
| AC2 | Registration form validates password strength (minimum 8 characters, mixed case, number, special char) | ✅ |
| AC3 | Registration form validates password confirmation matches password field | ✅ |
| AC4 | Backend rejects registration if email already exists for the same tenant | ✅ |
| AC5 | Upon successful registration, a new tenant is created with tenant_id (UUID) | ✅ |
| AC6 | User is assigned to the new tenant with a user_id (UUID) and username | ✅ |
| AC7 | User can immediately log in with registered email and password | ✅ |
| AC8 | Row-Level Security (RLS) is enforced: user can only access data for their own tenant_id | ✅ |

---

## Business Rules

- Email is unique per tenant (same email can exist in different tenants).
- Passwords must meet complexity requirements: minimum 8 characters, at least one uppercase, one lowercase, one number, one special character.
- Tenant is created atomically with the first user registration.
- No manual tenant creation endpoint exists; tenants are created on-demand during registration.
- Multi-tenant isolation is enforced at the database level via PostgreSQL RLS policies.

---

## Implementation Notes

- **Frontend:** `RegisterForm.tsx` in `features/auth/components/` with Zod schema validation.
- **Backend:** Auth registration endpoint in `AuthController`; `UserService` creates tenant and user atomically.
- **Database:** `users` and `tenants` tables with RLS enabled; `tenant_id` column on all core tables.
- **Tests:** `AuthIntegrationTest.java` validates registration flow; frontend tests for form validation.
