# US-102: Tenant Context & JWT

**Phase:** 1 — Core Load Lifecycle  
**Status:** COMPLETED  
**Depends On:** US-101  
**Blocks:** US-103, US-104, US-201, all downstream phases

---

## User Story

As a **registered user**, I need to receive JWT tokens (access + refresh) upon login so that my API requests are authenticated and automatically scoped to my tenant's data without manual tenant_id filtering.

---

## Acceptance Criteria

| ID | Criterion | Status |
|---|---|---|
| AC1 | Backend generates an access token (JWT) upon successful login that expires after 15 minutes | ✅ |
| AC2 | Backend generates a refresh token stored in an HTTP-only cookie that expires after 7 days | ✅ |
| AC3 | Frontend stores access token in Zustand in-memory state (never in localStorage) | ✅ |
| AC4 | JwtAuthenticationFilter validates the access token on every protected API request | ✅ |
| AC5 | Token validation fails (401 Unauthorized) if token is expired or signature is invalid | ✅ |
| AC6 | TenantContextHolder extracts tenant_id from JWT and injects it into all service-layer queries | ✅ |
| AC7 | All database queries automatically filter by tenant_id from TenantContextHolder context | ✅ |
| AC8 | Frontend can refresh the access token using the HTTP-only refresh cookie without re-entering credentials | ✅ |

---

## Business Rules

- Access token lifespan: 15 minutes (short-lived in-memory token).
- Refresh token lifespan: 7 days (long-lived HTTP-only cookie for session persistence).
- JWT audience must match the application (audience validation on parse).
- Tenant context must be set on every request before service-layer execution.
- Queries missing tenant_id filter will not execute (enforced at service layer via TenantContextHolder).
- Refresh tokens are single-use; a new token is issued on refresh.
- Token revocation is not implemented; rely on short lifespan for logout.

---

## Implementation Notes

- **Frontend:** `useLogin.ts` hook stores token in Zustand `authStore`; token added to all API requests via `apiClient.ts`.
- **Backend:** `JwtService` generates tokens; `JwtAuthenticationFilter` (93 lines) validates on every request; `TenantContextHolder` (51 lines) injects tenant context.
- **Security:** JWT signed with HS256 algorithm; refresh token stored in HttpOnly cookie via Spring Security.
- **Tests:** `JwtAuthenticationFilterTest.java`, `RefreshTokenServiceTest.java`, `TenantContextHolderTest.java`.
