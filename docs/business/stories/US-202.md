# US-202: In-App Notification Bell & Read Status

**Phase:** 2 — Notifications & EIA Integration  
**Status:** COMPLETED  
**Depends On:** US-201  
**Blocks:** US-301+

---

## User Story

As a **shipper or trucker**, I need to see an in-app notification bell that displays unread notifications so that I can quickly see the latest updates without leaving the app or checking email.

---

## Acceptance Criteria

| ID | Criterion | Status |
|---|---|---|
| AC1 | Notification bell appears in the top navigation bar of every page | ✅ |
| AC2 | Bell shows a badge with unread notification count (e.g., "5") | ✅ |
| AC3 | Clicking the bell opens a dropdown showing last 10 notifications | ✅ |
| AC4 | Each notification shows timestamp, load summary, and action (claim/pickup/delivery) | ✅ |
| AC5 | Notification is marked as read when clicked/viewed | ✅ |
| AC6 | Read notifications appear faded; unread notifications appear bold | ✅ |
| AC7 | New notifications appear in the bell in real-time (polled every 30 seconds or via SSE) | ✅ |
| AC8 | User can clear all notifications with a "Clear All" button | ✅ |

---

## Business Rules

- Notification list is paginated; "Load More" loads next 10 notifications.
- New notifications are polled from `/api/v1/notifications/unread` every 30 seconds.
- Real-time updates via Server-Sent Events (SSE) available for Phase 6 integration.
- Read status is persisted per notification per user.
- Clearing notifications sets `read_at = CURRENT_TIMESTAMP` for bulk update.
- Bell badge auto-updates after notification is read.

---

## Implementation Notes

- **Frontend:** `NotificationBell.tsx` in `features/notifications/components/`; Zustand store for unread count; React Query `useQuery` with 30-second refetch.
- **Backend:** `NotificationController` (GET /api/v1/notifications/unread, PATCH /api/v1/notifications/{id}/read, POST /api/v1/notifications/clear-all).
- **Database:** `notifications` table with `read_at` column; indexed on `user_id` and `read_at` for fast filtering.
- **Caching:** Unread notification count cached with 30-second TTL (NFR-504).
- **Tests:** `NotificationBellTest.tsx` (Vitest); `NotificationControllerTest.java` for API endpoints.
