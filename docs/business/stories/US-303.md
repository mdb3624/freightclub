# US-303: BOL/POD Photo Upload & Viewing

**Phase:** 3 — Document Management  
**Status:** COMPLETED  
**Depends On:** US-301  
**Blocks:** US-305

---

## User Story

As a **trucker or shipper**, I need to upload and view Bill of Lading (BOL) and Proof of Delivery (POD) photos so that I can submit required documentation and track delivery confirmation.

---

## Acceptance Criteria

| ID | Criterion | Status |
|---|---|---|
| AC1 | Trucker can upload BOL photo when load is CLAIMED (marks CLAIMED → ready for pickup) | ✅ |
| AC2 | Trucker can upload POD photo when load is IN_TRANSIT (marks IN_TRANSIT → DELIVERED) | ✅ |
| AC3 | Upload form validates file type (JPEG, PNG, WebP only) and size (<25 MB) | ✅ |
| AC4 | Uploaded photos are stored in S3 under load's document path | ✅ |
| AC5 | Shipper can view all uploaded BOL/POD photos from the load detail page | ✅ |
| AC6 | Each photo displays filename, upload timestamp, and uploader name | ✅ |
| AC7 | Photos are cached for 5 minutes to optimize load detail page performance | ✅ |
| AC8 | Upload completion is tracked; document list auto-refreshes after successful upload | ✅ |

---

## Business Rules

- BOL photos are required before trucker can mark load as picked up.
- POD photos are required before trucker can mark load as delivered.
- Multiple photos can be uploaded per load (e.g., multiple POD photos for multi-stop delivery).
- Shipper can view all photos; trucker can only view/upload their own photos.
- Photos are immutable once uploaded (no delete, overwrite only via new upload).
- Metadata stored in database; file content stored in S3.
- Cache TTL: 5 minutes (NFR-504 compliant).

---

## Implementation Notes

- **Frontend:** `DocumentSection.tsx` renders upload forms gated by load status and role.
- **Upload Hooks:** `useUploadBolPhoto(loadId)`, `useUploadPodPhoto(loadId)` fetch signed S3 URLs and handle upload.
- **Backend:** `DocumentController` (POST `/api/v1/loads/{id}/documents`, GET `/api/v1/loads/{id}/documents`); `DocumentService` manages metadata.
- **Database:** `load_documents` table tracks filename, upload_date, uploader_id, document_type, s3_key.
- **Caching:** Document list cached for 5 minutes; cache invalidated on new upload.
- **Tests:** `DocumentSectionTest.tsx` (Vitest) validates upload form; `DocumentControllerTest.java` tests API; E2E tests verify end-to-end flow.
