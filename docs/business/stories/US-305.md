# US-305: POD Upload UI Completion

**Phase:** 3 — Document Management
**Status:** COMPLETED
**Depends On:** US-301 (S3 File Storage), US-303 (BOL/POD Photo Upload)
**Blocks:** US-730 (Earnings Log), US-732 (IFTA Mileage), US-736 (Tax Export)

---

## User Story

As a **trucker**, I need to upload a Proof of Delivery (POD) photo after completing a delivery so that the shipper has documented confirmation and I can advance the load to DELIVERED status.

---

## Acceptance Criteria

| ID | Criterion | Status |
|---|---|---|
| AC1 | When a load is in CLAIMED status, the TRUCKER sees a BOL photo upload section | ✅ |
| AC2 | The "Mark as Picked Up" button is disabled until at least one BOL_PHOTO is uploaded | ✅ |
| AC3 | When a load is in IN_TRANSIT status, the TRUCKER sees a POD photo upload section | ✅ |
| AC4 | The "Mark as Delivered" button is disabled until at least one POD_PHOTO is uploaded | ✅ |
| AC5 | The TRUCKER can report a delivery issue with an optional photo while IN_TRANSIT | ✅ |
| AC6 | The SHIPPER sees an "Export PDF" button; the TRUCKER does not | ✅ |
| AC7 | Only JPEG, PNG, and WebP files are accepted for upload | ✅ |
| AC8 | Uploaded documents appear in the document list with filename, size, and timestamp | ✅ |
| AC9 | The upload button shows a loading state while the upload is in progress | ✅ |
| AC10 | After a successful upload, the document list refreshes automatically | ✅ |

---

## Business Rules

- A load cannot advance from CLAIMED → IN_TRANSIT without a BOL photo on record.
- A load cannot advance from IN_TRANSIT → DELIVERED without a POD photo on record.
- Document upload policy is enforced server-side by `LoadDocumentPolicy` (status validation).
- Maximum file size: 25 MB. Accepted types: JPEG, PNG, WebP.
- Multi-tenant isolation: documents are scoped to `tenant_id` via RLS.
