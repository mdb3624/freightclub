# User Story: US-308 — Document Audit Log Service

**Phase:** 3  
**Status:** IN_DEVELOPMENT  
**Depends On:** US-303 (BOL/POD Photo Upload & Viewing)  
**Blocks:** US-732 (IFTA Mileage Tracking), US-736 (Tax Compliance Export)  

---

## Overview

Implement the Document Audit Log Service to create immutable, timestamped records of all document lifecycle events (upload, download, signature, deletion attempts). This enables tax compliance reporting, dispute resolution, and regulatory audit trails.

---

## Acceptance Criteria

**AC-308-1:** Audit log entry created on document upload (S3 → `document_audit_log`)
- **When:** BOL or POD photo uploaded via S3
- **Then:** Insert record with `(document_id, user_id, tenant_id, action, timestamp, metadata)`
- **Verify:** Test `DocumentUploadService.logAuditEvent()` captures timestamp and user

**AC-308-2:** Audit log entry created on document download/view
- **When:** User downloads or views POD/BOL
- **Then:** Insert record with action = `DOWNLOAD` and viewer user_id
- **Verify:** Test `DocumentViewService` logs download event

**AC-308-3:** Audit log entry created on document signature
- **When:** Trucker signs POD in-app
- **Then:** Insert record with action = `SIGNED`, signature metadata, and timestamp
- **Verify:** Test `PodSignatureService.sign()` logs signature event with proof

**AC-308-4:** Immutable audit log (no updates, no deletes)
- **When:** Query `document_audit_log`
- **Then:** SELECT only; no UPDATE/DELETE allowed at SQL level
- **Verify:** PostgreSQL RLS policy prevents UPDATE/DELETE; Liquibase migration enforces

**AC-308-5:** Audit log supports 30-year retention (compliance)
- **When:** Archive/retention policy triggered
- **Then:** Data never deleted; soft-delete fields not used
- **Verify:** Schema has NO `deleted_at` on `document_audit_log`

**AC-308-6:** Multi-tenant isolation on audit queries
- **When:** Service queries audit log
- **Then:** Filter by `tenant_id = TenantContextHolder.getTenantId()`
- **Verify:** Integration test confirms cross-tenant data invisible

**AC-308-7:** Query audit trail for compliance reports
- **When:** User requests document audit trail
- **Then:** Service returns chronological list with all actions, users, timestamps
- **Verify:** Test `DocumentAuditService.getAuditTrail(documentId)` returns ordered records

**AC-308-8:** Cache audit trail query (5-minute TTL)
- **When:** `DocumentAuditService.getAuditTrail()` called
- **Then:** Cache key = `audit:{documentId}:{tenantId}`, TTL = 5m
- **Verify:** `@Cacheable` annotation present; `@CacheEvict` on new audit events

## Implementation Notes

### Database Schema (Flyway Migration)
```sql
CREATE TABLE document_audit_log (
  id VARCHAR(36) PRIMARY KEY,
  document_id VARCHAR(36) NOT NULL,
  user_id VARCHAR(36) NOT NULL,
  tenant_id VARCHAR(36) NOT NULL,
  action VARCHAR(50) NOT NULL, -- UPLOADED, DOWNLOADED, SIGNED, etc.
  metadata JSONB,
  created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (document_id) REFERENCES documents(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (tenant_id) REFERENCES tenants(id)
);

-- Immutable: no DELETE allowed
CREATE POLICY document_audit_log_no_delete ON document_audit_log
  AS (restrictive) FOR DELETE TO freightclub_runtime
  USING (false);

-- Multi-tenant isolation
CREATE POLICY document_audit_log_tenant_isolation ON document_audit_log
  AS (permissive) FOR SELECT TO freightclub_runtime
  USING (tenant_id = current_setting('app.tenant_id')::uuid);

CREATE INDEX idx_audit_document_created ON document_audit_log(document_id, created_at DESC);
CREATE INDEX idx_audit_tenant_created ON document_audit_log(tenant_id, created_at DESC);
```

### Service Layer
```java
// DocumentAuditService.java
@Service
public class DocumentAuditService {
  
  // Log new audit event
  public void logEvent(String documentId, String userId, String action, Map<String, Object> metadata) {
    // AC-308-1, AC-308-2, AC-308-3
    DocumentAuditLog entry = new DocumentAuditLog(
      UUID.randomUUID().toString(),
      documentId,
      userId,
      TenantContextHolder.getTenantId(),
      action,
      metadata,
      Instant.now()
    );
    auditRepository.save(entry);
    // Evict cache on new event (AC-308-8)
    cacheManager.getCache("audit").evict(documentId + ":" + TenantContextHolder.getTenantId());
  }
  
  // Query audit trail (AC-308-6, AC-308-7, AC-308-8)
  @Cacheable(
    value = "audit",
    key = "#documentId + ':' + T(com.freightclub.infrastructure.TenantContextHolder).getTenantId()",
    unless = "#result == null",
    cacheManager = "cacheManager"
  )
  public List<DocumentAuditLog> getAuditTrail(String documentId) {
    return auditRepository.findByDocumentIdAndTenantId(
      documentId,
      TenantContextHolder.getTenantId(),
      Sort.by("createdAt").descending()
    );
  }
  
  // AC-308-4: Repository only allows SELECT (RLS enforces)
  @Repository
  public interface DocumentAuditRepository extends JpaRepository<DocumentAuditLog, String> {
    List<DocumentAuditLog> findByDocumentIdAndTenantId(String documentId, String tenantId, Sort sort);
  }
}
```
