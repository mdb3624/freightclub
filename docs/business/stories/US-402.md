## Overview

Aggregate individual shipper ratings into a reputation score and profile. Display shipper reputation metrics (average rating, rating count, response time) to carriers browsing the load board, enabling carrier trust and filtering decisions.

---

## Acceptance Criteria

**AC-402-1:** Aggregate shipper ratings into average score
- **When:** User views shipper profile or load board
- **Then:** Calculate `avg(rating_value)` from all shipper's received ratings
- **Verify:** ShipperReputationService computes average; tests confirm correct calculation

**AC-402-2:** Shipper reputation includes rating count
- **When:** Shipper profile queried
- **Then:** Return count of total ratings received (denominator for avg)
- **Verify:** ShipperReputationResponse includes `totalRatingsReceived`

**AC-402-3:** Shipper reputation includes response time metrics
- **When:** Shipper reputation profile built
- **Then:** Calculate median/avg time from load publish â†’ shipper response
- **Verify:** ShipperReputationService aggregates load_published_at â†’ claim_created_at

**AC-402-4:** Cache shipper reputation (2-hour TTL)
- **When:** ShipperReputationService.getReputation() called
- **Then:** Cache key = `shipper_reputation:{shipperId}:{tenantId}`, TTL = 2h
- **Verify:** `@Cacheable` annotation present; test cache hit/miss

**AC-402-5:** Invalidate shipper reputation cache on new rating
- **When:** New rating created for shipper load
- **Then:** Evict cache for that shipper
- **Verify:** `@CacheEvict(ShipperReputationCacheInvalidator)` triggered on rating POST

**AC-402-6:** Multi-tenant isolation
- **When:** Service queries shipper reputation
- **Then:** Filter by `tenant_id = TenantContextHolder.getTenantId()`
- **Verify:** Integration test confirms cross-tenant reputations invisible

**AC-402-7:** Reputation profile includes metadata
- **When:** Shipper reputation returned
- **Then:** Include shipper name, company, location, rating distribution (1-5 star counts)
- **Verify:** ShipperReputationResponse includes all fields

## Implementation Notes

### Database Schema (Flyway Migration)
```sql
CREATE TABLE shipper_reputation (
  id VARCHAR(36) PRIMARY KEY,
  shipper_id VARCHAR(36) NOT NULL UNIQUE,
  tenant_id VARCHAR(36) NOT NULL,
  avg_rating DECIMAL(3,2),
  total_ratings_received INT DEFAULT 0,
  avg_response_time_minutes INT,
  median_response_time_minutes INT,
  last_rating_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (shipper_id) REFERENCES users(id),
  FOREIGN KEY (tenant_id) REFERENCES tenants(id)
);
```

### Domain Model
- `ShipperReputation` entity (domain)
- `ShipperReputationService` (application layer, caching, RLS-aware)
- `ShipperReputationCacheInvalidator` (cache eviction listener)
- `ShipperReputationRepository` (persistence)

### API Endpoints
- `GET /api/v1/shippers/{shipperId}/reputation` â€” View shipper reputation profile
- `GET /api/v1/loads` â€” Load board includes `shipper.reputation` embedded

### Caching Strategy
- Cache layer: Spring Cache (`ConcurrentHashMap` or Redis)
- Cache name: `shipper_reputation`
- TTL: 2 hours
- Eviction: Triggered on RatingService POST (new rating)
