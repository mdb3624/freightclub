# User Story: US-402 — Shipper Reputation Profile & Aggregation

**Phase:** 4  
**Status:** COMPLETED  
**Depends On:** US-401 (Bidirectional Rating System)  
**Blocks:** US-403 (Rating History & Timeline), US-405 (Shipper Reputation Badge on Load Board)  

---

## Overview

Aggregate shipper operational and payment behavior into a reliability profile. Display shipper reputation metrics (payment speed, completed/cancelled load counts, risk flags) to truckers browsing the load board, enabling informed trust decisions and load selection filtering. Track payment reliability, cancellation patterns, and dispute history.

---

## Acceptance Criteria

**AC-402-1:** Aggregate shipper payment reliability metrics
- **When:** User views shipper reputation profile
- **Then:** Calculate average payment speed (days from load delivery → payment confirmed) over last 90 days
- **Verify:** ShipperService computes `avg(payment_confirmed_at - delivered_at)`; tests confirm calculation

**AC-402-2:** Shipper reputation includes operational metrics
- **When:** Shipper profile queried
- **Then:** Return completed load count, cancelled load count (cancelled while CLAIMED), open dispute count
- **Verify:** ShipperReputationResponse includes all three metrics

**AC-402-3:** Shipper reputation includes risk flags
- **When:** Shipper reputation built
- **Then:** Calculate risk status: high-risk if >2 cancellations OR >2 open disputes; new-shipper if <3 completed loads
- **Verify:** ShipperReputation domain includes `hasHighRiskFlags()`, `isNewShipper()` methods

**AC-402-4:** Cache shipper reputation (2-hour TTL)
- **When:** ShipperService.getShipperReputation() called
- **Then:** Cache key = `shipperReputation:{shipperId}`, TTL = 2h
- **Verify:** `@Cacheable("shipperReputation")` annotation present; test cache hit/miss

**AC-402-5:** Invalidate shipper reputation cache on metric update
- **When:** Shipper reputation metrics updated via ShipperService.updateShipperReputation()
- **Then:** Evict cache for that shipper
- **Verify:** `@CacheEvict` triggered on reputation updates

**AC-402-6:** Multi-tenant isolation
- **When:** Service queries shipper reputation
- **Then:** Filter by `tenant_id = TenantContextHolder.getTenantId()` (via TenantContextHolder)
- **Verify:** Integration test confirms cross-tenant reputations invisible

**AC-402-7:** Reputation profile includes readable labels
- **When:** Shipper reputation returned
- **Then:** Include payment speed label (e.g., "Typically pays in 5 days"), risk indicators, shipper status
- **Verify:** ShipperReputationResponse and ShipperReputation domain include `getPaymentSpeedLabel()`, `hasHighRiskFlags()`

## Implementation Notes

### Database Schema (Flyway Migration)
```sql
CREATE TABLE shipper_reputation (
  id VARCHAR(36) PRIMARY KEY,
  shipper_id VARCHAR(36) NOT NULL UNIQUE,
  tenant_id VARCHAR(36) NOT NULL,
  average_payment_speed_days DECIMAL(5,1),
  completed_load_count INT DEFAULT 0,
  cancelled_load_count INT DEFAULT 0,
  open_dispute_count INT DEFAULT 0,
  last_calculated_at TIMESTAMPTZ NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMPTZ,
  FOREIGN KEY (shipper_id) REFERENCES users(id),
  FOREIGN KEY (tenant_id) REFERENCES tenants(id)
);
```

### Domain Model
- `ShipperReputation` entity (domain) with methods: `hasHighRiskFlags()`, `isNewShipper()`, `getPaymentSpeedLabel()`
- `ShipperService` (application layer, caching, RLS-aware, metrics aggregation)
- `ShipperReputationCacheInvalidator` (cache eviction listener)
- `ShipperReputationRepository` (persistence)
- `ShipperReputationEntity` (infrastructure mapping)

### API Endpoints
- `GET /api/v1/shippers/{shipperId}/public-reputation` — View shipper reputation profile (public, permitAll)
- Shipper reputation embedded in load board responses (trucker browsing loads)

### Caching Strategy
- Cache layer: Spring Cache (`ConcurrentHashMap` or Redis)
- Cache name: `shipperReputation`
- TTL: 2 hours
- Eviction: Triggered on ShipperService.updateShipperReputation() via @CacheEvict
