# User Story: US-403 — Rating History & Timeline

**Phase:** 4  
**Status:** COMPLETED  
**Depends On:** US-401 (Bidirectional Rating System)  
**Blocks:** US-405 (Shipper Reputation Badge on Load Board)  

---

## Overview

Provide users with a chronological rating history and timeline. Shippers and truckers can review all ratings they've received, when, from whom, and with what feedback. This supports dispute resolution and transparency in the platform's trust ecosystem.

---

## Acceptance Criteria

**AC-403-1:** Display user's received ratings in reverse chronological order
- **When:** User views their profile → Ratings tab
- **Then:** List all ratings received, newest first (created_at DESC)
- **Verify:** RatingController GET `/api/v1/users/{userId}/ratings` returns ordered list

**AC-403-2:** Rating history includes rater identity (anonymized or direct)
- **When:** Rating displayed
- **Then:** Show rater name/company (unless dispute escalated)
- **Verify:** RatingResponse includes `raterName` and `raterCompany`

**AC-403-3:** Rating history includes load context
- **When:** Rating viewed
- **Then:** Display load details: load ID, origin/destination, pickup date, delivered date
- **Verify:** RatingResponse includes `loadSummary` with contextual fields

**AC-403-4:** Rating comments displayed with rating value
- **When:** Rating includes comment
- **Then:** Display comment text alongside star rating
- **Verify:** RatingResponse.comment field is non-null and readable

**AC-403-5:** Cache rating history (30-minute TTL)
- **When:** RatingService.getUserRatings() called
- **Then:** Cache key = `user_ratings:{userId}:{tenantId}`, TTL = 30m
- **Verify:** `@Cacheable` annotation present; cache eviction on new ratings

**AC-403-6:** Rating timeline shows timestamps (RFC 3339 format)
- **When:** Rating viewed
- **Then:** created_at displayed as ISO 8601 (e.g., 2026-04-26T14:30:00Z)
- **Verify:** RatingResponse.createdAt serialized as RFC 3339

**AC-403-7:** Multi-tenant isolation
- **When:** User queries own rating history
- **Then:** Filter by `tenant_id = TenantContextHolder.getTenantId()`
- **Verify:** Integration test confirms users see only own tenant's ratings

**AC-403-8:** Pagination support for large rating histories
- **When:** User has >50 ratings
- **Then:** Support `?page=0&size=20` query params
- **Verify:** RatingController returns `Page<RatingResponse>` with metadata

## Implementation Notes

### Database Queries
```sql
-- User's received ratings (paginated)
SELECT * FROM load_ratings 
WHERE rater_id = ? AND tenant_id = ?
ORDER BY created_at DESC
LIMIT ? OFFSET ?;

-- Rating with load context (join)
SELECT r.*, l.origin, l.destination, l.pickup_date, l.delivery_date
FROM load_ratings r
JOIN loads l ON r.load_id = l.id
WHERE r.id = ? AND r.tenant_id = ?;
```

### API Endpoints
- `GET /api/v1/users/{userId}/ratings` — Get user's rating history (paginated)
- `GET /api/v1/ratings/{ratingId}` — View single rating with full context
- `GET /api/v1/loads/{loadId}/rating` — View rating for specific load

### Response Model
```json
{
  "id": "rating-123",
  "loadId": "load-456",
  "raterName": "John's Trucking",
  "raterRole": "TRUCKER",
  "ratingValue": 4.5,
  "comment": "Great communication, professional pickup",
  "createdAt": "2026-04-26T14:30:00Z",
  "loadSummary": {
    "origin": "Chicago, IL",
    "destination": "St. Louis, MO",
    "pickedUpAt": "2026-04-20T08:00:00Z",
    "deliveredAt": "2026-04-22T16:30:00Z"
  }
}
```

### Caching Strategy
- Cache name: `user_ratings`
- Cache key pattern: `user_ratings:{userId}:{tenantId}`
- TTL: 30 minutes
- Eviction: On new rating POST (any rating received by user)
