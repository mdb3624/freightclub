# User Story: US-405 — Shipper Reputation Badge on Load Board

**Phase:** 4  
**Status:** COMPLETED  
**Depends On:** US-402 (Shipper Reputation Profile & Aggregation)  
**Blocks:** None  

---

## Overview

Display a visual reputation badge next to shipper names on the load board. The badge provides at-a-glance trust signals (star rating, "verified shipper" status) to carriers browsing loads, reducing cognitive load and enabling quick filtering decisions.

---

## Acceptance Criteria

**AC-405-1:** Load board listing includes shipper reputation badge
- **When:** Carrier views load board
- **Then:** Each load row shows shipper name + reputation badge
- **Verify:** LoadSearchResponse includes `shipper.reputation.avgRating` and `shipper.reputation.totalRatings`

**AC-405-2:** Badge displays average rating (1-5 stars visual)
- **When:** Badge rendered
- **Then:** Show star rating (e.g., ★★★★☆ for 4.0)
- **Verify:** Frontend renders star icons proportional to avgRating

**AC-405-3:** Badge includes rating count context
- **When:** Badge displayed
- **Then:** Show count of ratings (e.g., "4.2 (28 ratings)")
- **Verify:** LoadSearchResponse.shipper.reputation includes `avgRating` and `totalRatingsReceived`

**AC-405-4:** Verified shipper indicator
- **When:** Shipper has ≥10 ratings and avg ≥4.0
- **Then:** Display "Verified Shipper" badge/checkmark
- **Verify:** ShipperReputationService calculates `isVerified` based on thresholds

**AC-405-5:** Badge color-codes reputation tier (visual salience)
- **When:** Badge rendered
- **Then:** Green (≥4.0), Yellow (3.0-3.9), Red (<3.0)
- **Verify:** Frontend applies Tailwind classes based on avgRating range

**AC-405-6:** Badge is clickable to view full reputation profile
- **When:** Carrier clicks badge
- **Then:** Navigate to `/shippers/{shipperId}/reputation` (detailed profile)
- **Verify:** Frontend routing configured; ShipperReputationController returns full data

**AC-405-7:** Badge respects shipper privacy (no personal data exposure)
- **When:** Badge displayed
- **Then:** Show only shipper name, company, avg rating, count
- **Verify:** LoadSearchResponse does NOT include shipper phone, email, address

**AC-405-8:** Badge honors multi-tenant isolation
- **When:** Shipper reputation badge fetched
- **Then:** Filter by `tenant_id = TenantContextHolder.getTenantId()`
- **Verify:** Integration test confirms cross-tenant reputations hidden

## Implementation Notes

### Frontend Component (React)
```tsx
// LoadCard.tsx
<div className="flex items-center gap-2">
  <span className="font-semibold text-gray-900">{load.shipper.name}</span>
  <ShipperReputationBadge 
    reputation={load.shipper.reputation}
    shipperId={load.shipper.id}
  />
</div>

// ShipperReputationBadge.tsx
const ShipperReputationBadge = ({ reputation, shipperId }) => {
  const badgeColor = 
    reputation.avgRating >= 4.0 ? 'bg-green-100 text-green-800' :
    reputation.avgRating >= 3.0 ? 'bg-yellow-100 text-yellow-800' :
    'bg-red-100 text-red-800';
  
  return (
    <Link to={`/shippers/${shipperId}/reputation`}>
      <div className={`px-2 py-1 rounded text-sm ${badgeColor}`}>
        ★ {reputation.avgRating.toFixed(1)} ({reputation.totalRatingsReceived})
        {reputation.isVerified && ' ✓'}
      </div>
    </Link>
  );
};
```

### Reputation Thresholds
- **Verified Shipper:** ≥10 ratings AND avgRating ≥4.0
- **Green Badge:** avgRating ≥4.0
- **Yellow Badge:** avgRating 3.0–3.9
- **Red Badge:** avgRating <3.0

### API Response Structure
```json
{
  "loads": [
    {
      "id": "load-123",
      "shipper": {
        "id": "shipper-456",
        "name": "ABC Shipping Co.",
        "reputation": {
          "avgRating": 4.2,
          "totalRatingsReceived": 28,
          "isVerified": true,
          "medianResponseTime": 15
        }
      }
    }
  ]
}
```

### Caching Strategy
- Load board search results cache includes shipper reputation (2h TTL)
- Shipper reputation cache (2h TTL) used to populate badge data
- Cache invalidation: On new rating POST → evict shipper reputation → load board cache stale-check
