# US-506: SETTLED Load Status & Workflow

**Status:** COMPLETED  
**Phase:** 5 (Payments & Invoicing)  
**Depends On:** US-502  
**Completed:** 2026-06-05  

---

## Summary

As a shipper, after a trucker marks a load as DELIVERED, I need to either confirm the delivery (transitioning to SETTLED) or file a dispute (transitioning to DISPUTED), so that payment can be released or withheld appropriately.

## Acceptance Criteria

- **AC-1:** Shipper can call `PATCH /api/v1/loads/{id}/settle` to transition a DELIVERED load to SETTLED; sets `settled_at` timestamp
- **AC-2:** Shipper can call `PATCH /api/v1/loads/{id}/dispute` with a `reason` body to transition a DELIVERED load to DISPUTED; sets `disputed_at` and `dispute_reason`
- **AC-3:** Both endpoints return 403 if the caller is not the load's owning shipper
- **AC-4:** Both endpoints return 409 if the load is not in DELIVERED status
- **AC-5:** `LoadDetailPage` shows a "Confirm Delivery" / "File Dispute" action block when load status is DELIVERED
- **AC-6:** Dispute form requires a non-empty reason before submission

## Business Rules

- Only shippers can settle or dispute a load
- A load must be in DELIVERED status to be settled or disputed
- Settle releases the load from shipper action queue (payment trigger deferred to US-510)
- Dispute freezes the load pending manual review

## Technical Implementation

**Backend:**
- `PATCH /api/v1/loads/{id}/settle` — `@PreAuthorize("hasRole('SHIPPER')")`
- `PATCH /api/v1/loads/{id}/dispute` — `@PreAuthorize("hasRole('SHIPPER')")`
- `DisputeRequest` record with `@NotBlank reason`
- `LoadSettledEvent` and `LoadDisputedEvent` domain events
- `settled_at TIMESTAMPTZ`, `disputed_at TIMESTAMPTZ`, `dispute_reason TEXT` columns added via `V20260605_1400__AddLoadSettledDisputedFields.sql`

**Frontend:**
- `loadsApi.settle(id)` / `loadsApi.dispute(id, reason)` in `api.ts`
- `useSettleLoad(loadId)` / `useDisputeLoad(loadId)` mutation hooks
- Inline confirm/dispute block in `LoadDetailPage.tsx` (amber callout, visible when status = DELIVERED)

**Tests:**
- `LoadServiceSettleDisputeTest.java` — 6 unit tests (settle success, settle wrong status, settle wrong tenant; dispute success, dispute wrong status, dispute wrong tenant)
