## User Story

**As a** *Trucker (owner/operator)*,  
**I want to** *maintain a profile describing my equipment inventory and preferred load lanes*,  
**so that** *the load board can recommend loads matching my capabilities and preferences.*

---

## Acceptance Criteria (Gherkin Style)

### AC-1: Trucker Can Add Equipment to Profile

**Given** a logged-in trucker is on the Profile → Equipment page  
**When** they click "Add Equipment" and enter:
- Equipment type (Flatbed, Dry Van, Refrigerated, Tanker, Specialized)
- Dimensions (length, width, height in feet)
- Capacity (weight in lbs)
- Equipment condition (Good, Fair, Needs Repair)
- Year/model (optional)

**Then** the system MUST:
- Validate equipment type is recognized
- Validate dimensions are positive numbers
- Create `carrier_equipment` record with:
  - `status = 'ACTIVE'`
  - `trucker_id` (from JWT context)
  - `tenant_id` (implicit, from TenantContextHolder)
  - `created_at`, `deleted_at = NULL`
- Return confirmation: "Equipment added to your profile."
- Display equipment in list with edit/delete options

**Example:**
```
Trucker adds:
  Type: Flatbed
  Dimensions: 48' x 8.5' x 6.5'
  Capacity: 45,000 lbs
  Condition: Good
  Year: 2022

System stores as single CarrierEquipment entity.
```

---

### AC-2: Trucker Can Set Preferred Load Lanes

**Given** a trucker is on the Profile → Preferred Lanes page  
**When** they click "Add Lane" and select:
- Origin region (state/multistate zone, e.g., "Southeast", "California", "Texas")
- Destination region (same options)
- Minimum rate per mile (optional, e.g., $1.50)
- Frequency preference (Daily, Weekly, Monthly, Any)

**Then** the system MUST:
- Create `carrier_lanes` record with:
  - `origin_region`, `destination_region`
  - `min_rate_cents` (optional)
  - `frequency_preference`
  - `status = 'ACTIVE'`
  - `tenant_id`, `trucker_id`, `created_at`
- Return confirmation: "Lane preference saved."
- Display lanes in list with ability to edit/deactivate
- Allow saving multiple lanes (e.g., Southeast→California, Texas→Northeast)

**Example:**
```
Trucker adds lane:
  Origin: Southeast
  Destination: California
  Min Rate: $1.75/mi
  Frequency: Weekly

System creates carrier_lanes entry for recommendations.
```

---

### AC-3: Trucker Can Set Availability Windows

**Given** a trucker is on the Profile → Availability page  
**When** they set:
- Days available (Mon–Fri, Weekends, or specific days)
- Hours available (time range, e.g., 08:00–17:00 EST)
- Time zone (EST, CST, MST, PST)
- Currently on load (Yes/No checkbox)

**Then** the system MUST:
- Create/update `carrier_availability` record with:
  - `available_days` (ENUM: MON_FRI, WEEKENDS, MON_SUN, CUSTOM)
  - `available_start_time`, `available_end_time` (HH:MM format)
  - `time_zone` (persisted for future reference)
  - `currently_on_load` (BOOLEAN)
  - `last_updated_at` (TIMESTAMPTZ)
- Return confirmation: "Availability updated."
- Display current availability status on dashboard

**Example:**
```
Trucker sets:
  Days: Mon–Fri
  Hours: 06:00–22:00 EST
  Currently on load: No

System allows board algorithm to check availability before suggesting loads.
```

---

### AC-4: Trucker Public Profile Viewable by Shippers

**Given** a shipper views a trucker's public profile (via link or search)  
**When** they click a trucker name on the load board or view their claim history  
**Then** the system MUST display (read-only):
- Trucker company name
- Star rating (from ratings system, Phase 4)
- Equipment inventory (equipment types, counts, conditions)
- Preferred lanes (origin→destination pairs)
- Load history summary (# completed, # current)
- Availability status (Available, On Load, Unavailable)
- Last seen (e.g., "Active 2 hours ago")

**DO NOT expose:**
- Bank account details
- Personal email/phone
- Insurance certificates
- Internal notes

**Example:**
```
Shipper views "John's Logistics" public profile:
  ⭐ 4.8/5 (87 ratings)
  
  Equipment:
    • Flatbed 48' (1 unit, Good condition)
    • Dry Van 53' (2 units, Good condition)
  
  Preferred Lanes:
    • Southeast → California ($1.75+/mi)
    • Texas → Northeast (any rate)
  
  History:
    • 156 loads completed
    • 1 currently active
  
  Status: Available (Mon–Fri, 06:00–22:00 EST)
  Last seen: 2 hours ago
```

---

### AC-5: Trucker Can Edit/Delete Equipment & Lanes

**Given** a trucker is viewing their equipment or lanes list  
**When** they click "Edit" or "Delete"  
**Then** the system MUST:
- For EDIT:
  - Pre-populate form with current values
  - Allow updating capacity, condition, dimensions
  - Save changes with `updated_at` timestamp
- For DELETE:
  - Soft-delete (set `deleted_at = CURRENT_TIMESTAMP`)
  - Remove from display immediately
  - Preserve history for audit trail
  - NOT allow deletion if currently claimed/active on load (error: "Cannot remove equipment in use")

**Example:**
```
Trucker edits equipment:
  Before: Flatbed 48' (Good) → After: Flatbed 48' (Needs Repair)
  System updates status, saves with updated_at timestamp.

Trucker deletes lane:
  Before: Southeast → California → After: deleted_at set
  Lane no longer appears in preferences or recommendations.
```

---

### AC-6: Multi-Tenancy & Data Isolation

**Given** any trucker interacting with profile data  
**When** they query or modify equipment/lanes/availability  
**Then** the system MUST:
- **ALWAYS** enforce `tenant_id = TenantContextHolder.current()`
- **NEVER** allow cross-tenant visibility of:
  - Equipment inventory
  - Preferred lanes
  - Availability windows
- **Use RLS Policy:** `SELECT/INSERT/UPDATE WHERE tenant_id = app.current_tenant_id AND deleted_at IS NULL`
- **Use PostgreSQL encryption** for sensitive fields (none required for public profile data)

**Example:**
```
Trucker from Tenant A queries their equipment:
  SELECT * FROM carrier_equipment WHERE tenant_id = ? AND deleted_at IS NULL
  
Trucker from Tenant B cannot see or modify Tenant A's equipment.
Public profile display uses RLS-filtered queries.
```

---

### AC-7: Load Board Uses Profile Data for Recommendations

**Given** the load board algorithm processes a new load  
**When** the load is posted by a shipper  
**Then** the system MUST:
- Query all active truckers' `carrier_lanes`, `carrier_equipment`, `carrier_availability`
- Match load against:
  - Equipment type required (load must match trucker's equipment)
  - Lane match (pickup region → delivery region matches trucker preference)
  - Rate match (load rate ≥ trucker's min_rate)
  - Availability match (current time within trucker's availability window)
- Create recommendations for matching truckers (persisted in `load_recommendations` table)
- Surface as "Suggested Loads" on trucker dashboard

**Example:**
```
New Load Posted:
  Type: Flatbed
  Origin: Atlanta (Southeast)
  Destination: Los Angeles (California)
  Rate: $2.00/mi
  Pickup: Tomorrow 06:00

Matching Trucker Profile:
  Equipment: Flatbed 48'
  Lanes: Southeast → California (min $1.75/mi)
  Available: Mon–Fri 06:00–22:00 EST
  Availability: Available (not on load)

Result: Load recommended to trucker.
```

---

### AC-8: Compliance & Audit Trail

**Given** any profile operation (add equipment, set lanes, update availability)  
**When** the operation completes  
**Then** the system MUST:
- Create immutable audit log entry in `carrier_profile_audit_log`:
  - `trucker_id`, `action` (ADD_EQUIPMENT, UPDATE_LANES, SET_AVAILABILITY, DELETE_EQUIPMENT)
  - `timestamp`, `ip_address`, `user_agent`
  - `data_before`, `data_after` (JSON for full audit trail)
  - `status_code` (SUCCESS, VALIDATION_ERROR, PERMISSION_ERROR)
- Log non-sensitive operations:
  - "Equipment added: Flatbed 48', 45000 lbs"
  - "Lane updated: Southeast→California, min $1.75/mi"
  - "Availability set: Mon-Fri 06:00–22:00 EST"
- **30-year retention:** Never delete audit logs

**Example:**
```
Audit Log:
  2026-04-28 14:30:00 | Trucker XYZ | ADD_EQUIPMENT | Flatbed 48' | SUCCESS
  2026-04-28 14:31:00 | Trucker XYZ | ADD_LANE | SE→CA, $1.75+/mi | SUCCESS
  2026-04-28 14:32:00 | Trucker XYZ | SET_AVAILABILITY | Mon-Fri 06:00–22:00 EST | SUCCESS
```

## Technical Context

### Domain Entities
1. **CarrierEquipment** (Aggregate Root)
   - Equipment type, dimensions, capacity, condition, year/model
   - Status (ACTIVE, INACTIVE)
   - Soft-delete support

2. **CarrierLane** (Value Object / Separate Table)
   - Origin region, destination region, min rate, frequency
   - Status (ACTIVE, INACTIVE)
   - Soft-delete support

3. **CarrierAvailability** (Value Object / Separate Table)
   - Available days, hours, time zone
   - Currently on load (boolean)
   - Last updated timestamp

4. **CarrierProfileAuditLog** (Immutable Ledger)
   - All profile operations tracked
   - Data before/after for audit

### Dependencies
- **Database:** PostgreSQL with RLS policies
- **Ratings System:** Phase 4 (for star ratings on public profile)
- **Load Board Algorithm:** Phase 7 Suggested Loads feature
- **User Context:** TenantContextHolder, JWT authentication

### Non-Functional Requirements
- **Latency:** Profile updates must complete within 500ms
- **Availability:** Profile service 99.9% available
- **Consistency:** RLS isolation enforced on all queries
- **Auditability:** 30-year retention of all profile operations
