## User Story

**As a** *Trucker*,  
**I want to** *see loads recommended by the board based on my equipment and lane preferences*,  
**so that** *I can quickly find relevant loads without manually searching through the entire board.*

---

## Acceptance Criteria

### AC-1: Load Board Algorithm Matches Equipment & Lanes

**Given** a new load is posted to the board  
**When** the system processes it  
**Then** it MUST match against:
- Equipment type required (load.equipmentType ∈ trucker.equipment[])
- Lane preferences (load.originRegion → load.destinationRegion ∈ trucker.lanes[])
- Rate match (load.ratePerUnit ≥ trucker.minRate OR trucker.minRate is NULL)
- Availability (current time ∈ trucker.availabilityWindow)
- Trucker not already on load (trucker.currentlyOnLoad = false)

**Result:** Create `load_recommendation` record (trucker_id, load_id, match_score)

---

### AC-2: Trucker Views Suggested Loads Dashboard

**Given** a logged-in trucker  
**When** they click "Suggested Loads" on the load board  
**Then** the system displays:
- List of recommended loads (newest first)
- Each load shows: origin, destination, rate, equipment type, delivery deadline
- Match reason: "Matches your Flatbed preference, SE→CA lane"
- Action buttons: "View Details", "Claim Load"
- Filter/sort options (by rate, deadline, distance)

**Example:**
```
Suggested Loads (8 matches)

1. Atlanta → Los Angeles | $2,100 | Flatbed | Due: Tomorrow 14:00
   ✓ Matches: Flatbed, SE→CA lane, $2.00/mi rate
   [View Details] [Claim Load]

2. Jacksonville → Phoenix | $1,800 | Flatbed | Due: Tomorrow 18:00
   ✓ Matches: Flatbed, SE→SW lane, $1.85/mi rate
   [View Details] [Claim Load]
```

---

### AC-3: Recommendation Algorithm Scores Matches

**Given** a load and trucker with overlapping preferences  
**When** calculating recommendation score  
**Then** score MUST be:
- Exact equipment match: +100 points
- Lane preference match: +50 points
- Rate above trucker minimum: +25 points
- Availability match: +25 points
- Final score displayed to trucker (1-200 scale, visual indicator: 🟢 Perfect / 🟡 Good / 🔴 Fair)

**Algorithm:**
```
score = 0
score += (load.equipmentType in trucker.equipment) ? 100 : 0
score += (load.route in trucker.lanes) ? 50 : 0
score += (load.rate >= trucker.minRate) ? 25 : 0
score += (load.pickup_time in trucker.availability) ? 25 : 0
matchQuality = score >= 150 ? "Perfect" : score >= 100 ? "Good" : "Fair"
```

---

### AC-4: Recommendations Update in Real-Time

**Given** a trucker claims a load or updates availability  
**When** the action completes  
**Then** the system MUST:
- Remove that load from recommendations (claimed)
- Recalculate availability window
- Update all previous recommendations if availability changed
- Refresh "Suggested Loads" list on trucker dashboard

---

### AC-5: Multi-Tenancy & Data Isolation (RLS)

**Given** the recommendation algorithm runs  
**When** matching trucker profiles against loads  
**Then** MUST:
- Only match loads within trucker's tenant
- Only match against trucker's own equipment & lanes
- RLS query: `WHERE load.tenant_id = ? AND equipment.tenant_id = ?`
- No cross-tenant recommendations

---

### AC-6: Audit & Compliance

**Given** a recommendation is generated  
**When** the load_recommendation record is created  
**Then** record MUST include:
- load_id, trucker_id, match_score, match_reason (JSON)
- created_at timestamp
- Immutable ledger (never updated, soft delete only)

## Technical Context

### Domain Entity
```
LoadRecommendation (Value Object)
- loadId, truckerId, matchScore (1-200)
- matchReason { equipment: bool, lane: bool, rate: bool, availability: bool }
- createdAt, deletedAt (soft delete)
```

### Algorithm Flow
1. Load posted → trigger recommendation job
2. Query truckers with matching equipment types
3. Filter by lane preferences (load.origin → load.destination)
4. Filter by rate (load.rate >= trucker.minRate)
5. Filter by availability (load.pickupTime ∈ trucker.availability)
6. Score each match (0-200)
7. Create load_recommendation records
8. Push notification to recommended truckers (optional)

### Database Changes
- New `load_recommendations` table
- Indexes: (load_id), (trucker_id), (created_at)
- RLS policy: tenant isolation
