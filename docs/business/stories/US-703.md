# US-703: Shipper Preferred Carrier Management

**Phase:** Phase 7 (Advanced Carrier Management)  
**Status:** READY_FOR_DESIGN  
**Owner:** BA (Michael)  
**Dependency:** US-701 (Carrier Profiles) ✅ COMPLETED  
**Traceability:** REQ-7.6, REQ-7.7 (Preferred Carrier Networks)

---

## User Story

**As a** *Shipper*,  
**I want to** *save a list of preferred truckers and directly assign loads to them, bypassing the open board*,  
**so that** *I can build trusted relationships with reliable carriers and ensure load coverage.*

---

## Acceptance Criteria

### AC-1: Shipper Can Save Preferred Trucker

**Given** a shipper is viewing a trucker's public profile  
**When** they click "Add to Preferred"  
**Then** the system MUST:
- Create `preferred_carriers` record (shipper_id, trucker_id, tenant_id)
- Record `added_at` timestamp
- Display confirmation: "Trucker added to your preferred list"
- Show "Remove from Preferred" button (toggle state)

---

### AC-2: Shipper Can View Preferred Carrier List

**Given** a shipper is on Carrier Management page  
**When** they click "Preferred Carriers"  
**Then** display:
- List of all preferred truckers (with status: Online, Offline, On Load)
- Each trucker: name, rating, equipment count, last load, availability status
- Actions: "Direct Assign Load", "View Profile", "Remove from Preferred"
- Search/filter by name or rating

---

### AC-3: Shipper Can Directly Assign Load to Preferred Trucker

**Given** a shipper has a new load and a list of preferred truckers  
**When** they click "Direct Assign Load" from the preferred list  
**Then** the system MUST:
- Show a modal with equipment/lane requirements
- Match against selected trucker's profile
- If compatible, create load claim (status = DIRECT_ASSIGNED, not OPEN_CLAIM)
- Send notification to trucker: "New load directly assigned"
- Mark load as claimed (status = CLAIMED)
- Log in audit trail: "Direct assigned to {trucker_name}"

**Example:**
```
Direct Assign Load
Selected Trucker: John's Logistics (4.8⭐)
Equipment Match: ✓ Flatbed (has 1)
Lane Match: ✓ SE→CA (preferred)
Rate: $2,000 (min: $1,500)

[Assign Load] [Cancel]
```

---

### AC-4: Shipper Can Block Carriers

**Given** a shipper has had issues with a specific trucker  
**When** they click "Block Carrier" (on trucker profile or preferred list)  
**Then** the system MUST:
- Create `blocked_carriers` record (shipper_id, trucker_id, blocked_at)
- That trucker NO LONGER sees shipper's loads in the open board
- No future load recommendations from shipper to that trucker
- Shipper cannot direct assign to blocked carriers
- Display "Unblock" option if needed

---

### AC-5: Load Claiming Rules Enforce Direct Assignment

**Given** a shipper has directly assigned a load  
**When** another trucker tries to claim it  
**Then** the system MUST:
- Return error: "This load is not available (reserved for another carrier)"
- Log attempt in audit trail
- Do NOT allow override/force claim

---

### AC-6: Multi-Tenancy & Data Isolation

**Given** any preferred carrier operation  
**When** shippers query or modify preferred lists  
**Then** MUST:
- Enforce `tenant_id = TenantContextHolder.current()`
- Shipper A cannot see Shipper B's preferred list
- RLS policy: `SELECT/INSERT/UPDATE WHERE tenant_id = app.current_tenant_id`

---

### AC-7: Audit Trail

**Given** any preferred carrier action (add, remove, block, unblock, direct assign)  
**When** operation completes  
**Then** record in `preferred_carrier_audit_log`:
- action (ADD, REMOVE, BLOCK, UNBLOCK, DIRECT_ASSIGN)
- shipper_id, trucker_id, load_id (if applicable)
- timestamp, ip_address, user_agent

---

## Technical Context

### Domain Entities
```
PreferredCarrier (Shipper → Trucker relationship)
- id, shipper_id, trucker_id, tenant_id
- added_at, deleted_at (soft delete)

BlockedCarrier (Shipper blocks Trucker)
- id, shipper_id, trucker_id, tenant_id
- blocked_at, unblocked_at (soft delete)

LoadClaim (modified to support direct assignment)
- claim_type: OPEN_CLAIM | DIRECT_ASSIGNED
- assigned_by_shipper: boolean
```

### Database Changes
- New `preferred_carriers` table
- New `blocked_carriers` table
- Modify `load_claims` to include `claim_type` and `direct_assigned_by_shipper_id`
- New `preferred_carrier_audit_log` table
- Indexes: (shipper_id, trucker_id), (shipper_id, tenant_id)
- RLS policies on all tables

### Algorithm
```
When load posted:
  if shipper has direct_assigned = trucker:
    set load.status = CLAIMED
    prevent other truckers from claiming
  else:
    include in open board
    generate recommendations (US-702)
```
