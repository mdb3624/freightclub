## User Story

**As an** *Admin or Shipper*,  
**I want to** *view analytics on load board activity, including posting trends, matching success rates, and market demand*,  
**so that** *I can optimize pricing, timing, and carrier selection strategies.*

---

## Acceptance Criteria

### AC-1: Admin Views Load Board Analytics Dashboard

**Given** an admin is logged in  
**When** they navigate to Analytics > Load Board  
**Then** display:
- Total loads posted (last 7/30/90 days)
- Total loads claimed (percentage claimed vs. unclaimed)
- Average time-to-claim (in hours)
- Peak posting hours (histogram)
- Top lanes by volume (origin → destination)
- Equipment type distribution (pie chart)

**Example:**
```
Load Board Analytics (Last 30 Days)
────────────────────────────────
Total Posted:      1,247 loads
Claimed:           892 (71%)
Avg Time-to-Claim: 3.2 hours

Top Lanes:
1. Atlanta → Los Angeles   (142 loads)
2. Dallas → Chicago        (128 loads)
3. Houston → New York      (115 loads)

Peak Hours: 8-9am (ETZ), 2-3pm (ETZ)
Equipment: Flatbed (45%), Dry Van (38%), Tanker (12%), Specialized (5%)
```

---

### AC-2: Shipper Views Performance Insights for Their Loads

**Given** a shipper is viewing their load history  
**When** they click "Performance Insights"  
**Then** display:
- Average claim time for their loads
- Success rate (claimed vs. posted)
- Average match score distribution (1-200)
- Preferred carrier claim percentage
- Pricing tier performance (e.g., loads at $1.50/mi vs. $2.00/mi)

**Example:**
```
My Load Performance (Last 30 Days)
──────────────────────────────────
Posted:                   47 loads
Claimed:                  44 (94%)
Avg Claim Time:          1.8 hours
Avg Match Score Claimed: 162/200 (Good)

Claimed by Preferred:     28 (64%)
Claimed by Open Board:    16 (36%)

Price Analysis:
$1.50/mi:  12 posted, 10 claimed (83%)
$2.00/mi:   8 posted,  8 claimed (100%)
```

---

### AC-3: Recommendation Matching Insights

**Given** a load is posted  
**When** recommendations are generated  
**Then** system records and displays:
- Number of truckers matched
- Average match score for matched truckers
- Match reason breakdown (equipment, lane, rate, availability counts)
- Percentage of matched truckers who claimed vs. didn't claim

**Example:**
```
Load: Atlanta → LA (Flatbed, $2,100)
──────────────────────────────────────
Recommendations Generated: 47 truckers
Avg Match Score:          156/200

Match Reasons:
- Equipment match:  47 (100%)
- Lane match:       38 (81%)
- Rate match:       45 (96%)
- Availability:     42 (89%)

Claim Conversion:  9 clicked, 3 claimed (7% → 6%)
```

---

### AC-4: Demand Forecast by Lane & Equipment

**Given** admin is planning capacity  
**When** they view Forecast > Demand  
**Then** display:
- Historical load posting trends (7/30/90 day moving average)
- Forecast for next 7/14/30 days
- Seasonal patterns (if enough historical data)
- Equipment demand forecast (which equipment types trending up/down)

**Example:**
```
Demand Forecast (Next 30 Days)
────────────────────────────────
Historical Avg:  42 loads/day
Forecasted:      48 loads/day (+14%)

Top Growing Lanes:
1. Southeast → West Coast   (+22%)
2. Midwest → Northeast      (+18%)

Equipment Trends:
- Flatbed:    +8% demand (seasonal spring)
- Tanker:     -3% (post-winter maintenance)
- Specialized: +15% (specialty cargo season)
```

---

### AC-5: Multi-Tenancy & Data Isolation

**Given** any analytics query  
**When** admin/shipper requests data  
**Then** MUST:
- Return only data for their tenant
- Aggregate metrics only within tenant scope
- Enforce `tenant_id` filter on all queries
- RLS policy: `SELECT WHERE tenant_id = app.current_tenant_id`

---

### AC-6: Performance & Caching

**Given** heavy analytics queries (large datasets)  
**When** queries are executed  
**Then** MUST:
- Cache aggregations (daily, hourly buckets)
- Return results in < 2 seconds
- Use materialized views or pre-computed summaries
- Background job recalculates every 1 hour (or on-demand)

---

### AC-7: Export & Reporting

**Given** an admin wants external reporting  
**When** they click "Export Analytics"  
**Then** allow:
- CSV export (all data for date range)
- PDF summary report (formatted dashboard)
- Email scheduled reports (daily/weekly summary)

## Technical Context

### Domain Entities
```
LoadAnalytics (Immutable, Append-Only)
- loadId, tenantId, postedAt, claimedAt (nullable)
- matchCount, avgMatchScore, matchReasons (JSON)
- claimTime (seconds), claimedBy (trucker_id or null)

LaneAnalytics (Aggregated, Pre-Computed)
- origin, destination, tenantId
- postCount, claimCount, avgPrice, avgClaimTime
- computedAt (timestamp)

EquipmentDemand (Aggregated, Pre-Computed)
- equipmentType, tenantId, date (daily bucket)
- postCount, claimCount, avgClaimTime
- computedAt
```

### Database Changes
- New `load_analytics` table (append-only, soft delete)
- New `lane_analytics_aggregated` table (pre-computed, refreshed hourly)
- New `equipment_demand_daily` table (pre-computed, refreshed hourly)
- Indexes on (tenant_id, date), (lane_origin_dest), (equipment_type)
- RLS policies on all tables

### Algorithm
```
When load claimed:
  INSERT into load_analytics(loadId, tenantId, claimTime, matchScore...)
  TRIGGER: refresh_lane_analytics_aggregated() -- update counts
  TRIGGER: refresh_equipment_demand_daily() -- update equipment trends

Background Job (Hourly):
  RECALCULATE lane_analytics_aggregated FROM load_analytics
  RECALCULATE equipment_demand_daily FROM load_analytics
  RECALCULATE demand_forecast_7day FROM historical trends
```
