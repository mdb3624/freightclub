**Status:** READY_FOR_DESIGN

---

## User Story

**As a** *Shipper*,  
**I want to** *view performance metrics for carriers (acceptance rate, on-time delivery, quality scores)*,  
**so that** *I can identify the most reliable carriers and negotiate contracts with data.*

---

## Acceptance Criteria

### AC-1: Shipper Views Carrier Performance Summary

**Given** a shipper is viewing a carrier's profile  
**When** they click "Performance Metrics"  
**Then** display:
- Acceptance rate (% of assigned loads accepted)
- On-time delivery rate (% loads delivered by promised time)
- Average delivery time vs. estimated
- Load completion count (last 30/90 days)
- Quality score (1-5 stars based on shipper ratings)
- Preferred by X shippers (count)

**Example:**
```
Performance: John's Logistics
──────────────────────────────
Acceptance Rate:      94% (142/151 assigned)
On-Time Delivery:     89% (125/140 completed)
Avg Delivery Time:    2.1 days (est. 2.0)
Loads Completed:      140 (last 90 days)
Quality Rating:       ⭐⭐⭐⭐⭐ 4.8/5 (23 ratings)
Preferred By:         12 shippers
```

---

### AC-2: Acceptance Rate Tracking

**Given** a carrier is assigned a load  
**When** they accept, decline, or miss the deadline  
**Then** system tracks:
- Assignment timestamp
- Accept/decline action (and timestamp)
- Reason for decline (if provided)
- Assignment → accept time (in hours)
- Rolling 30/90-day acceptance rate

**Example:**
```
Assignment History (Last 30 Days)
──────────────────────────────────
Assigned:  151 loads
Accepted:  142 (94%)
Declined:   7 (5%) — Reason: "out of service area"
Missed:     2 (1%) — No response in 2hr window

Accept Time: Avg 0.5 hours, Median 0.3 hours
Trend:       +2% vs. previous 30 days
```

---

### AC-3: On-Time Delivery Metrics

**Given** a load is in-transit  
**When** delivery is completed  
**Then** system records:
- Scheduled delivery time
- Actual delivery time
- On-time status (delivered by promise time)
- Early/late variance (in hours)
- Delivery proof timestamp

**Example:**
```
Delivery Performance (Last 30 Days)
───────────────────────────────────
Completed Loads: 140
On-Time:        125 (89%)
Early:           8 (5%)
Late:            7 (6%) — Avg 2.3 hours late

Variance Analysis:
- Early: 2.1 hours avg
- Late:  2.3 hours avg
- On-time rate trend: +3% vs. previous 30 days
```

---

### AC-4: Quality Ratings & Feedback

**Given** a shipper receives a delivery  
**When** they rate the carrier  
**Then** system allows:
- 1-5 star rating
- Optional feedback text
- Specific criteria: professionalism, equipment condition, communication
- Rating is immutable (audit trail)

**Example:**
```
Rate This Carrier: John's Logistics
───────────────────────────────────
Overall:         ⭐⭐⭐⭐⭐ (5 stars)
Professionalism:  Excellent
Equipment:        Clean & Well-Maintained
Communication:    Responsive & Clear

Feedback: "Great driver, delivered early, very professional."
```

---

### AC-5: Shipper-Specific Performance Trends

**Given** a shipper views a carrier they've worked with  
**When** clicking "My History with This Carrier"  
**Then** display:
- Number of loads assigned to this carrier (all-time)
- Completion rate (specific to this shipper)
- Average claim time
- Cost per unit (if shipper negotiated rate)
- Rating history (last 5 ratings by this shipper)

**Example:**
```
My History: John's Logistics
────────────────────────────
Total Loads Assigned: 47
Completed:            44 (94%)
Pending:              2 (4%)
Cancelled:            1 (2%)

Avg Claim Time:      0.6 hours
Negotiated Rate:     $1.95/mi (contract)
Actual Cost (avg):   $1.93/mi
My Ratings:          5★, 5★, 4★, 5★, 5★
```

---

### AC-6: Industry Benchmarking

**Given** a shipper is viewing carrier metrics  
**When** they want context  
**Then** display:
- Peer comparison (industry average for acceptance, on-time rate)
- Carrier percentile ranking (e.g., "top 15% for on-time delivery")
- Trend vs. platform average

**Example:**
```
How Does This Carrier Compare?
──────────────────────────────
Acceptance Rate:   94% (Carrier) vs. 87% (Platform Avg) ✅ +7%
On-Time Delivery:  89% (Carrier) vs. 84% (Platform Avg) ✅ +5%
Quality Rating:    4.8★ (Carrier) vs. 4.2★ (Platform Avg) ✅ +0.6

Percentile Ranking: Top 18% for overall reliability
```

---

### AC-7: Multi-Tenancy & Data Isolation

**Given** any performance query  
**When** shipper/admin requests data  
**Then** MUST:
- Show only carriers within shipper's tenant
- Show only loads assigned by this shipper
- Ratings visible only to relevant shipper (not cross-tenant)
- RLS policy: `SELECT WHERE tenant_id = app.current_tenant_id`

## Technical Context

### Domain Entities
```
CarrierPerformance (Immutable, Audit-Trail)
- carrierId, tenantId, recordedAt
- loadAssigned (count), loadAccepted (count), loadDeclined (count)
- acceptanceRate (%), onTimeRate (%), avgDeliveryTime
- qualityScore (1-5 stars), ratingCount

ShipperCarrierRating (Immutable, Immutable)
- carrierId, shipperId, tenantId, ratedAt
- stars (1-5), feedback (text), criteria (JSON)
- canDelete = false (audit only)

CarrierMetricsSummary (Pre-Computed, Refreshed)
- carrierId, tenantId, period (daily/weekly/monthly)
- acceptanceRate, onTimeRate, avgDeliveryTime
- computedAt (timestamp)
```

### Database Changes
- New `carrier_performance` table (append-only)
- New `carrier_metrics_summary` table (pre-computed, refreshed daily)
- New `shipper_carrier_ratings` table (immutable audit)
- Indexes: (carrier_id, tenant_id), (shipper_id, carrier_id)
- RLS policies on all tables
