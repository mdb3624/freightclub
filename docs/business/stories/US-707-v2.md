## User Story

**As a** Shipper,
**I want to** search for registered carriers by name or email and add them to my preferred list,
**so that** I can build a trusted carrier network without guessing carrier IDs.

---

## Acceptance Criteria

**AC-v2-1: Page has navigation header**
- **When** Shipper navigates to `/settings/preferred-carriers`
- **Then** The standard FreightClub nav bar renders (FreightClub logo, My Loads, Profile links, user name, Sign out)
- **Verify:** `AppShell` wraps `PreferredCarriersList`

**AC-v2-2: Carrier search returns real results**
- **When** Shipper types 2+ characters in the search field
- **Then** `GET /api/v1/carriers/search?q={term}` is called (debounced 300ms)
- **Then** Results show up to 8 TRUCKER users from the same tenant matching by first name, last name, or email
- **Verify:** Results include carrier name, email, equipment type badge

**AC-v2-3: Empty search state**
- **When** Search returns no results
- **Then** Show `data-testid="carrier-search-empty"` with message `No carriers found for "{term}"`

**AC-v2-4: Carrier selection**
- **When** Shipper clicks a result
- **Then** Selected carrier chip appears with name, email, teal checkmark; search input clears
- **Then** `Add to Preferred List` button becomes enabled
- **Verify:** `data-testid="selected-carrier-chip"` visible

**AC-v2-5: Clear selection**
- **When** Shipper clicks the ✕ on the selected chip
- **Then** Selection clears, search input returns, button re-disables
- **Verify:** `data-testid="clear-carrier-selection"` triggers reset

**AC-v2-6: Tenant isolation**
- **When** Carrier search runs
- **Then** Only TRUCKER users in the same `tenant_id` are returned
- **Verify:** `TenantContextHolder.getTenantId()` is used as the filter

**AC-v2-7: Min length guard**
- **When** Search query is < 2 characters
- **Then** No API call is made; hint text shown
- **Verify:** Service returns empty list for queries < 2 chars
