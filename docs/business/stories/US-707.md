# User Story: US-707 — Shipper Preferred Carrier List

**Phase:** 7  
**Status:** READY_FOR_DESIGN  
**Owner:** BA  
**Depends On:** US-101 (Multi-Tenant Registration)  
**Blocks:** US-708 (Direct Load Assignment to Carrier)  
**Traceability:** REQ-7.1 (Carrier Network Management)

---

## User Story

**As a** *Shipper*,  
**I want to** *maintain a list of preferred trucking carriers and manage which carriers can see my loads*,  
**so that** *I can build trusted partnerships, negotiate better rates, and control load distribution across my preferred network.*

---

## Acceptance Criteria

**AC-707-1:** Shipper can add carriers to preferred list
- **When:** Shipper navigates to Settings > Preferred Carriers
- **Then:** Display a form to add carrier by email/ID + optional notes (e.g., "negotiated 10% discount")
- **Verify:** POST /api/v1/shippers/preferred-carriers accepts `(carrierId, notes)`, stores in `shipper_preferred_carriers` table

**AC-707-2:** Shipper can view preferred carriers list
- **When:** Shipper opens Preferred Carriers page
- **Then:** Display table with carrier name, email, date added, notes, remove button
- **Verify:** GET /api/v1/shippers/preferred-carriers returns paginated list (sorted by date added DESC)

**AC-707-3:** Shipper can remove carriers from preferred list
- **When:** Shipper clicks remove on a carrier row
- **Then:** Confirm "Remove this carrier from preferred list?" → DELETE carrier
- **Verify:** DELETE /api/v1/shippers/preferred-carriers/{carrierId} soft-deletes record

**AC-707-4:** Cache preferred carrier list (1-hour TTL)
- **When:** GET /api/v1/shippers/preferred-carriers called
- **Then:** Cache key = `preferredCarriers:{shipperId}:{tenantId}`, TTL = 1h
- **Verify:** `@Cacheable("preferredCarriers")` present; test cache hit/miss

**AC-707-5:** Invalidate cache on add/remove
- **When:** Carrier added or removed from preferred list
- **Then:** Evict cache for shipper
- **Verify:** `@CacheEvict` triggered on POST/DELETE

**AC-707-6:** Multi-tenant isolation
- **When:** Service queries preferred carriers
- **Then:** Filter by `tenant_id = TenantContextHolder.getTenantId()`
- **Verify:** Shipper A cannot see Shipper B's preferred carriers

**AC-707-7:** Preferred carrier count visible on dashboard
- **When:** Shipper views dashboard
- **Then:** Show badge "X Preferred Carriers" (e.g., "12 Preferred Carriers")
- **Verify:** Widget calls GET /api/v1/shippers/preferred-carriers?limit=1 to fetch count

## Implementation Notes

### Database Schema (Flyway Migration)
```sql
CREATE TABLE shipper_preferred_carriers (
  id VARCHAR(36) PRIMARY KEY,
  shipper_id VARCHAR(36) NOT NULL,
  carrier_id VARCHAR(36) NOT NULL,
  tenant_id VARCHAR(36) NOT NULL,
  notes TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMPTZ,
  FOREIGN KEY (shipper_id) REFERENCES users(id),
  FOREIGN KEY (carrier_id) REFERENCES users(id),
  FOREIGN KEY (tenant_id) REFERENCES tenants(id),
  UNIQUE(shipper_id, carrier_id, deleted_at IS NULL)
);

CREATE INDEX idx_shipper_preferred_carriers ON shipper_preferred_carriers(shipper_id, deleted_at);
```

### Domain Model
- `ShipperPreferredCarrier` entity (domain)
- `ShipperPreferredCarrierService` (application layer, caching, RLS-aware)
- `ShipperPreferredCarrierRepository` (persistence)
- Exception: `CarrierAlreadyPreferredException`, `CarrierNotFoundException`

### API Endpoints
- `POST /api/v1/shippers/preferred-carriers` — Add carrier
- `GET /api/v1/shippers/preferred-carriers` — List preferred carriers (paginated)
- `DELETE /api/v1/shippers/preferred-carriers/{carrierId}` — Remove carrier
- `GET /api/v1/shippers/preferred-carriers/count` — Get count only (for dashboard badge)

### Caching Strategy
- Cache name: `preferredCarriers`
- TTL: 1 hour
- Key: `{shipperId}:{tenantId}`
- Eviction: Triggered on add/remove
