# US-709: Block/Restrict Carrier

**Phase:** Phase 7 (Carrier Management)  
**Status:** READY_FOR_DESIGN  
**Owner:** BA  
**Dependency:** US-701 (Carrier Profile) ✅ COMPLETED  
**Traceability:** REQ-7.3 (Carrier Governance)

---

## User Story

**As a** *Shipper*,  
**I want to** *block specific carriers from being assigned my loads*,  
**so that** *I can prevent problematic carriers from servicing my freight while maintaining my preferred carrier network.*

---

## Acceptance Criteria

### AC-1: Shipper Blocks a Carrier

**Given** a shipper is viewing a carrier profile  
**When** they click "Block This Carrier" and provide a reason  
**Then** system:
- Records the block with timestamp
- Prevents future automatic assignment to that carrier
- Keeps block indefinite until shipper unblocks
- Displays confirmation: "Carrier blocked successfully"

---

### AC-2: View Blocked Carriers List

**Given** a shipper is in their settings  
**When** they navigate to "Blocked Carriers"  
**Then** display:
- Paginated table (20 per page) of blocked carriers
- Columns: Carrier ID, Reason (if provided), Blocked Date, Actions
- Total count badge showing number of blocked carriers
- Empty state message if no carriers blocked

---

### AC-3: Unblock a Carrier

**Given** a shipper has blocked carriers  
**When** they click "Unblock" on a carrier  
**Then** system:
- Soft-deletes the block record (deletedAt timestamp)
- Carrier becomes eligible for future assignments
- Shows confirmation: "Carrier unblocked"

---

### AC-4: Check Block Status

**Given** system is assigning a load  
**When** shipper has blocked a carrier  
**Then** system:
- Queries: `isCarrierBlocked(shipperId, carrierId)`
- Returns true/false
- Prevents assignment to blocked carriers

---

### AC-5: Multi-Tenancy

**Given** any block operation  
**When** block is recorded  
**Then** MUST:
- Isolate by tenant_id
- Only show shipper's own blocked carriers
- Enforce RLS: `WHERE tenant_id = app.current_tenant_id`

---

## Technical Context

### Domain Entity
```
BlockedCarrier
- id (UUID)
- shipperId (UUID)
- carrierId (UUID)
- tenantId (UUID)
- reason (TEXT, nullable)
- createdAt (TIMESTAMPTZ)
- deletedAt (TIMESTAMPTZ, soft delete)
```

### Database
- New `blocked_carriers` table
- Unique constraint: (tenant_id, shipper_id, carrier_id)
- Index on (tenant_id, shipper_id, deletedAt)
- RLS enabled
