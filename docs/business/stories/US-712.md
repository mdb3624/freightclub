# US-712: View Shipper Public Profile (Payment Speed, Rating)

**Phase:** 7b (Financial Intelligence)  
**Status:** MIGRATION_PENDING  
**Depends On:** US-102 (Ratings), US-502 (Quick Pay / Payment Status)  
**Blocks:** US-702 (Preferred Lanes — shipper trust influences lane selection)

---

## User Story

> As a trucker, I want to see a shipper's payment speed, rating, and reputation indicators before claiming a load, so I can assess their reliability and avoid loads from shippers with poor payment histories.

---

## Business Context

Truckers make claim decisions partly based on shipper reputation. A shipper with a history of slow or disputed payments, or frequent load cancellations, is less attractive. The platform should surface this data on load cards and accessible via a shipper profile to:
1. Incentivize shippers to maintain good payment practices
2. Help truckers make informed decisions
3. Build trust in the marketplace

## Acceptance Criteria

### AC-1: Shipper Profile Modal Accessible from Load Card
- When trucker clicks on shipper name on a load card, a modal dialog opens showing shipper's public profile
- Modal is readable and dismissible via close button or backdrop click
- Modal displays immediately without loading spinner (data should be cached per AC-7)

### AC-2: Profile Displays Core Reputation Signals
- **Overall Rating:** 1–5 star badge (or "New shipper" if < 3 completed loads)
- **Average Payment Speed:** Displayed as human-readable text (e.g., "Typically pays in 7 days") or "Not yet rated" if no completed loads
- **Completed Load Count:** Total number of loads delivered and settled (credibility indicator for new shippers)
- **Dispute/Cancellation Flag:** Visual warning indicator (amber triangle/icon) if shipper has:
  - Cancelled >2 CLAIMED loads (stranding truckers), OR
  - Open/flagged payment disputes (3+ in last 90 days)

### AC-3: Average Payment Speed Calculation
- **Formula:** `avg(payment_confirmed_at − delivered_at)` across all completed loads in the last 90 days
- **Display Format:** "Typically pays in N days" (rounded to nearest day; e.g., "Typically pays in 7 days")
- **Null Handling:** If shipper has 0 completed loads, display: "New shipper — no rating yet"
- **Null Handling:** If shipper has completed loads but zero payments processed, display: "No completed payments"

### AC-4: New Shipper Badge
- If shipper has < 3 completed loads, display "🆕 New shipper" badge instead of a numeric rating
- This prevents truckers from downrating shippers with insufficient track record

### AC-5: Dispute/Cancellation Warning Indicator
- Display as amber warning triangle or cautionary icon if:
  - Shipper cancelled >2 loads while in CLAIMED status (active load was freed), OR
  - Shipper has >2 open/flagged payment disputes in last 90 days
- Include tooltip/hover text: "This shipper has a history of [cancellations / payment disputes]"
- Do NOT permanently ban shippers; warning is visible but loadable

### AC-6: Shipper Profile Page Link
- Modal includes a "View Full Profile" button linking to shipper's full public profile page at `/shippers/:shipper_id`
- Full profile shows extended history, past loads, all ratings, detailed payment timeline

### AC-7: Data Cached 1 Hour (NFR-504)
- Shipper reputation data cached with 1h TTL
- Cache invalidated on:
  - New payment confirmed (reduces cache by cache key)
  - New rating submitted by trucker (invalidates shipper cache)
  - Shipper cancels a load (invalidates shipper cache)

## Technical Notes

### Backend Requirements
- New query: `getShipperPublicReputation(shipper_id, now)` → {rating, avg_payment_speed, load_count, cancellation_flags, dispute_flags}
- Query must respect multi-tenancy (shipper's loads visible cross-tenant as public data)
- Average payment speed calculation SQL:
  ```sql
  SELECT AVG(EXTRACT(DAY FROM (payments.confirmed_at - loads.delivered_at)))
  FROM loads
  JOIN payments ON loads.id = payments.load_id
  WHERE loads.shipper_id = ?
    AND loads.delivered_at >= NOW() - INTERVAL '90 days'
    AND payments.confirmed_at IS NOT NULL
    AND loads.status = 'settled'
  ```

### Frontend Requirements
- Shipper name on load cards links to `<ShipperProfileModal shipper_id={load.shipper_id} />`
- Modal fetches from `/api/v1/shippers/{shipper_id}/public-reputation` (cached)
- Reusable modal component for load cards and shipper profile pages
- Display "Loading..." while data fetches; cache hit should be instant

### Permission Model
- Shipper reputation is PUBLIC (visible to all authenticated truckers)
- Shipper name/email visible ONLY after trucker claims the load (existing behavior)
- Public profile accessible ONLY if trucker has claimed a load from that shipper OR via shipper name on load card
