# US-752: Lazy-Load Font Subsets After Authentication

**Phase:** Infrastructure (Performance Optimization)  
**Status:** DESIGN_APPROVED  
**Priority:** CRITICAL  
**Effort:** 2-3 hours  
**Impact:** 40% reduction in initial page load (200-300 KB deferred)

---

## User Story

As a **platform operator**, I need to defer font loading until after authentication so that the login page loads with system fonts instead of waiting for 500+ KB of custom font files, reducing initial page load time by 40%.

## Business Rules

- Login/register pages must never wait for font loading
- System font stack must be legible and acceptable for unauthenticated users
- Dashboard must load full font subsets (no degradation for authenticated users)
- Font loading must not block JavaScript execution
- No Flash of Unstyled Text (FOUT) acceptable during transition
- Font subsets should include Latin only for login page

---

## Technical Design

**Current state:**
- All 24 font files (500+ KB) loaded in `<head>`
- Fonts load synchronously, blocking page render

**Target state:**
- Login page: System fonts only (0 KB custom fonts)
- Dashboard: Full font subsets loaded asynchronously after auth
- Fonts cached locally by browser (not re-downloaded on auth)

**Implementation:**
- Create system font fallback in CSS
- Move custom fonts to lazy load bundle
- Trigger font load in `AuthInitializer` component
