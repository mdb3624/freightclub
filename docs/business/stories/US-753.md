# US-753: Replace Zod Validation with Lightweight Regex for Login

**Phase:** Infrastructure (Performance Optimization)  
**Status:** DESIGN_APPROVED  
**Priority:** HIGH  
**Effort:** 1-2 hours  
**Impact:** 7% reduction in auth bundle (36-42 KB savings)

---

## User Story

As a **platform operator**, I need to replace the Zod validation library (28-32 KB) with inline regex validation for the login form since only 2-3 validation rules are used, eliminating unnecessary library overhead.

## Business Rules

- Login form has only 2 fields: email + password
- Email validation: required + valid email format
- Password validation: required only
- Error messages must match current UX
- No other forms in auth module use complex validation

---

## Technical Design

**Replace:**
```typescript
// OLD: Zod + resolver
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'

const schema = z.object({
  email: z.string().min(1, 'Email is required').email('Enter a valid email'),
  password: z.string().min(1, 'Password is required'),
})
```

**With:**
```typescript
// NEW: Inline regex validation
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
const validateForm = (data) => {
  const errors = {}
  if (!data.email) errors.email = 'Email is required'
  else if (!emailRegex.test(data.email)) errors.email = 'Enter a valid email'
  if (!data.password) errors.password = 'Password is required'
  return errors
}
```
