# US-754: Optimize Cloud CDN Configuration for Asset Delivery

**Phase:** Infrastructure (Performance Optimization)  
**Status:** DESIGN_APPROVED  
**Priority:** MEDIUM  
**Effort:** 1-2 hours  
**Impact:** 10-15% faster repeated page loads

---

## User Story

As a **platform operator**, I need to configure Cloud CDN with proper cache headers, Brotli compression, and TTL settings so that assets are cached globally and repeated visits load 10-15% faster.

---

## Acceptance Criteria

| ID | Criterion | Status |
|---|---|---|
| AC1 | Hashed assets (*.js, *.css) have 1-year cache headers (immutable) | ⏳ |
| AC2 | index.html has 1-hour cache with revalidation (must-revalidate) | ⏳ |
| AC3 | Brotli compression enabled for JS/CSS (in addition to gzip) | ⏳ |
| AC4 | Cloud CDN TTL set to 3600+ seconds for static assets | ⏳ |
| AC5 | Repeated page loads show cache hits in DevTools | ⏳ |
| AC6 | Cache headers verified via curl for all asset types | ⏳ |
| AC7 | Brotli compression reduces JS by additional 15-20% vs gzip | ⏳ |
| AC8 | No regression in Cloud Run health checks | ⏳ |

---

## Business Rules

- Hashed assets (*.abc123.js) never change, cache forever
- index.html changes frequently, revalidate frequently
- Brotli compression must fall back to gzip for older browsers
- Cloud CDN must be enabled on Load Balancer
- All compression transparent to users

---

## Technical Design

**Cache headers by asset type:**
- `*.js` (hashed): `Cache-Control: public, max-age=31536000, immutable`
- `*.css` (hashed): `Cache-Control: public, max-age=31536000, immutable`
- `index.html`: `Cache-Control: public, max-age=3600, must-revalidate`
- Fonts: `Cache-Control: public, max-age=604800, immutable`

**Compression:**
- Enable Brotli in Cloud Run response headers
- gzip fallback for compatibility

**Cloud CDN:**
- Enable on Load Balancer (may already be enabled)
- Set TTL to 3600+ seconds
- Enable cache key customization

## Testing Strategy

```bash
curl -I https://freightclub-frontend.run.app/assets/index*.js | grep Cache-Control
curl -I --compressed https://freightclub-frontend.run.app/assets/index*.js | grep Content-Encoding
```

**Lighthouse report:**
- Compare before/after cache header changes
- Verify repeated visit improvement
