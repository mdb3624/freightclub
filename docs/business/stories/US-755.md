# US-755: Replace React Query for Static Dashboard Queries

**Phase:** Infrastructure (Performance Optimization)  
**Status:** DESIGN_APPROVED  
**Priority:** MEDIUM  
**Effort:** 3-4 hours  
**Impact:** 5-8% additional bundle reduction (5-8 KB)

---

## User Story

As a **platform operator**, I need to replace React Query usage for static queries (available states, diesel prices, profile) with simple Axios + useState patterns since these queries don't benefit from React Query's caching/refetching features.

## Business Rules

- Static queries: useAvailableStates, useDieselPrices
- Semi-dynamic: useProfile (per-session, not refetched)
- Dynamic queries: useLoadBoard, useMyActiveLoad, useMyLoadHistory (keep React Query)
- No functionality change from user perspective
- Session-scoped caching acceptable (no cross-session persistence needed)

---

## Technical Design

**Replace Pattern:**

```typescript
// OLD: React Query (uses 38-42 KB for all 6 hooks)
const { data: states } = useAvailableStates()

// NEW: Axios + useState
const [states, setStates] = useState<AvailableStates | null>(null)
const [isLoading, setIsLoading] = useState(true)
useEffect(() => {
  if (!states) {
    loadApi.getAvailableStates()
      .then(setStates)
      .finally(() => setIsLoading(false))
  }
}, [])
```

**Hooks to replace:**
1. `useAvailableStates` → fetch + useState
2. `useDieselPrices` → fetch + useState  
3. `useProfile` → evaluate (likely keep as React Query if used elsewhere)

**Hooks to keep (React Query):**
- `useLoadBoard` → pagination, filtering, frequent refetch
- `useMyActiveLoad` → real-time updates needed
- `useMyLoadHistory` → pagination
- Custom mutations (claim, publish, cancel)
