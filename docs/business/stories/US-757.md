## User Story

As a **trucker/owner-operator**, I need to track granular operating costs (fuel, maintenance, insurance, permits, per diem) so that I can accurately calculate my minimum acceptable rate per mile (RPM) and make informed decisions about which loads to accept.

---

## Business Context

Truckers currently lack visibility into their true operating costs. Without accurate cost tracking:
- They cannot calculate minimum RPM (revenue per mile needed to break even + profit margin)
- They cannot evaluate load profitability quickly
- They accept unprofitable loads unknowingly
- They cannot optimize for better-paying lanes or load types

This story enables the **Cost Per Mile (CPM) Calculator**, a tool that calculates:
- Fixed Cost Per Mile (fixed costs ÷ monthly miles)
- Variable Cost Per Mile (fuel + maintenance costs)
- Total Cost Per Mile (fixed + variable)
- **Minimum RPM = Total CPM + Target Profit Margin**

## Acceptance Criteria

### AC1: Granular Cost Input - Fixed Monthly Costs
**Scenario:** Trucker sets up cost profile with individual fixed cost components  
**Given:** Trucker is on the profile/settings page  
**When:** Trucker enters values for:
- Truck Payment/Lease ($)
- Insurance ($)
- IFTA/IRP/Permits ($)
- Phone/ELD/Misc ($)  
**Then:** 
- All four fields are saved to the profile
- Fields can be edited individually
- Subtotal is calculated: `Truck Payment + Insurance + IFTA + Phone/ELD`

### AC2: Variable Cost Input - Fuel & Maintenance
**Scenario:** Trucker tracks fuel and maintenance costs  
**Given:** Trucker is on the profile/settings page  
**When:** Trucker enters:
- Diesel Price ($/Gal)
- Fuel Efficiency (MPG)
- Maintenance Reserve ($/Mile)  
**Then:** 
- All three fields are saved
- Fuel CPM calculated: `Diesel Price ÷ MPG`
- Variable CPM calculated: `Fuel CPM + Maintenance CPM`

### AC3: Per Diem Tracking
**Scenario:** Trucker includes per diem in daily operating costs  
**Given:** Trucker is on the profile/settings page  
**When:** Trucker enters:
- Per Diem Daily Rate ($)
- Days per Month working  
**Then:** 
- Both fields are saved
- Monthly per diem calculated: `Daily Rate × Days/Month`
- Included in total fixed costs

### AC4: Operational Targets
**Scenario:** Trucker sets miles and profit targets  
**Given:** Trucker is on the profile/settings page  
**When:** Trucker enters:
- Monthly Miles Driven (target)
- Target Profit Margin per Mile ($)  
**Then:** 
- Both fields are saved
- Used in Minimum RPM calculation

### AC5: Automatic CPM Calculation & Display
**Scenario:** System calculates and displays cost breakdown  
**Given:** Trucker has entered at least one cost value and monthly miles  
**When:** Trucker views the cost profile form  
**Then:** A "Calculated Cost Breakdown" section displays:
- Fixed CPM = Total Fixed Costs ÷ Monthly Miles
- Variable CPM = Fuel CPM + Maintenance CPM
- **Total CPM** = Fixed + Variable
- **Minimum RPM** = Total CPM + Target Profit Margin  
**And:** Values update in real-time as trucker edits inputs

### AC6: Form Layout & Organization
**Scenario:** Cost profile is organized into logical sections  
**Given:** Trucker is on the profile/settings page  
**Then:** Form displays three sections with clear headers:
1. **Fixed Monthly Costs** (4 fields: truck, insurance, IFTA, phone/ELD)
2. **Variable Costs** (5 fields: diesel, MPG, maintenance, per diem daily rate, per diem days)
3. **Operational** (2 fields: monthly miles, profit margin)

### AC7: Optional Fields with Smart Defaults
**Scenario:** Trucker can leave fields blank without errors  
**Given:** Any cost field is empty  
**When:** Calculation occurs  
**Then:** 
- Empty fields are treated as 0
- RPM calculation shows partial results (e.g., if miles=0, shows "Enter monthly miles" instead of infinity)
- No validation errors on blank fields

### AC8: Data Persistence
**Scenario:** Cost profile is saved to backend  
**Given:** Trucker completes cost profile form and navigates away  
**When:** Trucker returns to the profile page  
**Then:** All previously entered values are restored

### AC9: API Contract
**Scenario:** Frontend and backend share consistent field names  
**Then:** Profile API (`PUT /api/v1/profile`) accepts and returns:

  "insurance": 900,
  "iftaIrpPermits": 200,
  "phoneEldMisc": 150,
  "perDiemDailyRate": 800,
  "perDiemDaysPerMonth": 20,
  "fuelCostPerGallon": 3.89,
  "milesPerGallon": 6.5,
  "maintenanceCostPerMile": 0.17,
  "monthlyMilesTarget": 8000,

## Business Rules

1. **Cost Breakdown:** Fixed costs are tracked individually, not combined.
2. **Per Diem Calculation:** Automatically computed from daily rate × days/month.
3. **All Fields Optional:** Trucker can enter partial data; missing fields default to 0.
4. **Real-time CPM:** Calculator updates instantly as trucker edits values.
5. **Minimum RPM Formula:** `Total CPM + Target Profit Margin per Mile`
6. **Data Scope:** Each trucker sees only their own cost profile (multi-tenant isolation).
7. **Input Validation:** Backend validates cost field ranges before persistence:
   - All cost fields (truck payment, insurance, IFTA, phone/ELD, per diem rate, fuel price, maintenance): **≥ 0** (or null)
   - Per diem days per month: **1–31** (or null)
   - Monthly miles target: **> 0** (or null)
   - Invalid input returns 400 BAD_REQUEST with error details.
   - Empty/null fields are allowed and treated as 0 per AC7.
