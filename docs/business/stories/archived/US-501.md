## User Story

**As a** *Trucker (user role)*,  
**I want to** *receive settlement payment within 2-3 business days OR opt for Quick Pay (accelerated payout)*,  
**so that** *I have predictable cash flow and the option to accelerate payout when needed (at a small fee).*

## Acceptance Criteria (Gherkin Style)

### AC-1: Standard Settlement (2-3 Business Days) â€” Zero Fee

**Given** a load has transitioned to DELIVERED status  
**When** the load automatically transitions to SETTLED (or admin approves settlement)  
**Then** the system MUST:
- Calculate gross amount = `load.rate_per_unit Ã— load.quantity_value` (in cents)
- Deduct 2% platform commission (or tenant override rate)
- Create three immutable transactions atomically:
  - `LOAD_SETTLEMENT` (gross amount)
  - `PLATFORM_COMMISSION` (2% deducted)
  - `TRUCKER_PAYOUT` (net amount, 98% of gross)
- Record `QUICK_PAY_FEE: $0.00`
- Schedule ACH payout for 2-3 business days out
- Display estimated payout date on trucker dashboard

**Example:**
```
Load rate: $1,000.00
Platform commission (2%): $20.00
Trucker net payout: $980.00
Quick Pay fee (standard): $0.00
Final payout: $980.00 (in 2-3 business days)
```

### AC-2: Quick Pay Election (Next Business Day) â€” 1% Fee

**When** the load transitions to SETTLED

  - `PLATFORM_COMMISSION` (2% deducted)
  - `QUICK_PAY_FEE` (1% of base payout, additional fee for acceleration)
  - `TRUCKER_PAYOUT` (base payout âˆ’ quick pay fee)
- Schedule payout for next business day (by 5pm EST)
- Display "Next Business Day (by 5pm)" on trucker dashboard

**Example:**
```

Platform commission (2%): $20.00
```

### AC-4: Financial Ledger Immutability

**Given** a settlement is complete  
**When** any party queries `financial_transactions` table  
**Then** the system MUST:
- Ensure all transaction records are immutable (no UPDATE on amount_cents, transaction_type, commission_rate_percent)
- Allow only soft-delete (status = 'CANCELLED') for voided transactions
- Enforce RLS: `WHERE tenant_id = current_tenant_id`
- Return complete audit trail with timestamps and created_by
- Provide reconciliation view: `SUM(LOAD_SETTLEMENT) âˆ’ SUM(PLATFORM_COMMISSION) âˆ’ SUM(TRUCKER_PAYOUT) = 0` (balance check)

---

### AC-5: Tenant Commission Override Support

**Given** a tenant has a custom commission rate (e.g., 1.75% for volume discount)  
**When** settlement is calculated  
**Then** the system MUST:
- Look up active `tenant_commission_overrides` for the tenant
- Apply custom rate (or default 2.0% if no override)
- Record the exact rate applied in the `commission_rate_percent` field
- Prospectively apply (new settlements only; no retroactive changes)

**Example (with 1.75% override):**
```
Load rate: $1,000.00
Platform commission (1.75%): $17.50
Base payout: $982.50
Quick Pay fee (1% of base): $9.83
Final payout (Quick Pay): $972.67
```

---

### AC-6: Payout Method Configuration

**Given** a trucker has configured a payment account (ACH/Wire)  
**When** settlement creates a payout  
**Then** the system MUST:
- Retrieve trucker's configured payment method
- Create `payment_intent` record with:
  - `gross_amount_cents`, `net_amount_cents`, `platform_fee_cents`
  - `payment_method` (ACH, WIRE, etc.)
  - `status = PENDING`
- Use payment processor to initiate transfer (scope: Phase 5, Task 2)

---

### AC-7: Dispute Hold on Contested Settlements

**Given** a shipper files a dispute BEFORE settlement is finalized  
**When** settlement is triggered  
**Then** the system MUST:
- Check for active disputes on the load
- If dispute exists: create `DISPUTE_HOLD` transaction, hold payout pending resolution
- If no dispute: proceed with normal settlement
- Allow admin to resolve dispute and release/forfeit funds

## Technical Constraints

### Database Design
- **Primary Table:** `financial_transactions` (immutable, append-only)
- **Schema Requirements:**
  - `id` (VARCHAR(36), PK)
  - `tenant_id` (VARCHAR(36), not null, RLS policy)
  - `load_id` (VARCHAR(36), FK to loads)
  - `transaction_type` (ENUM: LOAD_SETTLEMENT, PLATFORM_COMMISSION, QUICK_PAY_FEE, DISPUTE_HOLD, DISPUTE_RESOLUTION, REFUND)
  - `amount_cents` (BIGINT, no negatives)
  - `commission_rate_percent` (DECIMAL(5,2), audit trail)
  - `shipper_id`, `trucker_id` (VARCHAR(36), FKs for auditability)
  - `status` (ENUM: PENDING_APPROVAL, COMPLETED, REVERSED, DISPUTED, CANCELLED)
  - `created_at`, `completed_at`, `created_by` (audit trail)
  - Indexes: `(tenant_id, created_at)`, `(load_id)`, `(transaction_type, status)`
