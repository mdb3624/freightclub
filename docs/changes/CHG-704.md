## CHG-704: E2E Playwright Evidence Requires Running App Stack

**Original Story:** US-UI-MIGRATION (Theme State UI Component Migration)
**Discovered By:** REVIEWER on 2026-06-07
**Root Cause:** REVIEWER hard gate requires a Playwright golden-path E2E run with a `page.screenshot()` artifact in `test-results/evidence/` for the story. Producing real evidence requires the components to be wired into a routed page AND the full Docker stack (backend + frontend + Postgres + auth seeding) running per `testing_standards.md`. That integration (route registration, page composition, Docker reset/build cycle) is materially larger than "implement theme-state UI components" and was not scoped by BA/HFD as part of this story's AC.
**Technical Blocker:** Cannot produce a real, non-fabricated screenshot artifact without standing up the full stack — fabricating evidence would violate [[feedback_gate_verification_before_signoff]].

**Options:**
1. Approve this story's component-level deliverables now (unit tests green, HFD spec satisfied) and track the E2E golden-path spec + screenshot as a follow-up story that also covers route/page wiring.
2. Block this story until the full Docker E2E stack is started and a real Playwright run produces the screenshot (large scope increase, blocked on Docker/infra availability).
3. Defer the entire UI migration to Phase 8 pending dedicated E2E infra time.

**Recommendation:** Option 1 — unit-level TDD evidence is real and green; the missing piece is integration wiring + infra execution, which is better scoped as its own story (`US-UI-MIGRATION-E2E`) so it gets proper BA/HFD/ARCH sign-off on page composition.

**Decision:** LIBRARIAN: "Finish US-UI-MIGRATION with Option 1. PR notes CHG-704 tracked separately. Create backlog story US-UI-MIGRATION-E2E for route wiring + Playwright golden-path evidence."
