## CHG-858: Home Page Fidelity Gaps vs. `handoff/` Design Package (Footer Contact Info, Modal A11y) — RESOLVED

**Original Story:** US-855 (Marketing Home Page & In-Page Login Modal) — COMPLETED
**Discovered By:** User-directed re-verification of `frontend/src/pages/HomePage.tsx` against a resent design handoff (`handoff/`), 2026-07-22. Byte-for-byte diff (ignoring line-endings) confirmed `handoff/` is the same design package already implemented for US-855 — no new visual/copy delta. Re-verification proceeded as a spec-compliance audit of the live implementation rather than a new build.

**Root Cause:** Two real gaps between the live `HomePage.tsx`/`LoginModal.tsx` and the handoff spec, found via automated agent audit + confirmed by reading the source:

1. **Dead "Contact Us" link.** `HomePage.tsx` footer rendered `<a href="#">Contact Us</a>` with no real contact info. The handoff's `CLAUDE_CODE_PROMPT.md` (section 6) explicitly specifies a footer "Contact Us" column: Mike Barnes, (404) 960-9621, mike.barnes@mdbfreightclub.com, with tel:/mailto: links. Never implemented.
2. **Login modal missing dialog accessibility semantics.** `LoginModal.tsx` had no `role="dialog"`, `aria-modal`, `aria-labelledby`, no Escape-to-close handler, and no focus management on open. `handoff/README.md` line 40 ("No console errors; accessible form labels/roles on the modals (focus trap, ESC to close, aria attributes)") requires this; it's also a genuine accessibility defect independent of the handoff spec.

**Not a gap (verified against documented scope):** the login modal has no in-modal "create account" view — it links out to `/register`. Confirmed against US-855's own "Out of Scope" section, which explicitly defers a dedicated registration modal. Handoff docs describe an aspirational full build; actual story scope overrode it deliberately.
