## CHG-862: Two More E2E Root Causes — Broken `networkidle` Wait and Missing Mobile-Menu Handling

**Discovered By:** Follow-on from CHG-861 — after the auth-race fix, real CI evidence showed E2E timing out at both 15m and 25m job budgets regardless of worker count, with tests consistently dying at their own `test.setTimeout` ceiling rather than a global hang. Root-caused via a full local 125-test run (109 passed / 13 failed / 3 skipped in 38.3 minutes — proving no true deadlock, just genuine per-test failures plus real slowness).

**Root Cause 1 — `networkidle` on a continuously-polling page.** `US-844-carrier-load-board.spec.ts`'s `loginAsCarrier` helper called `page.waitForLoadState('networkidle')` after navigating to `/dashboard/trucker`. `TruckerDashboard.tsx` polls live data (diesel prices, load board) continuously, so network is never idle and the wait never resolves — every affected test hung to its 90s `test.setTimeout` ceiling. The correct fix (wait for a concrete rendered element) was already known and correctly applied in `US-841-ui-primitives.spec.ts` with an explanatory comment; it just wasn't applied consistently. Fixed by mirroring that pattern: wait for `page.getByTestId('trucker-dashboard')` instead.

**Root Cause 2 — login helpers don't handle the mobile hamburger menu.** `header-login-btn` lives inside `<nav className="hidden ... md:flex">` (`HomePage.tsx`) — genuinely not rendered below Tailwind's `md` (768px) breakpoint. The fallback selector many login helpers used, `header-get-started-btn-mobile`, doesn't exist anywhere in the current header markup (stale, likely from an earlier design). At any narrow viewport (320px/375px, used throughout the "adversarial" test blocks), the combined locator matched nothing and the click just hung until timeout. Fixed 14 call sites across 8 files to check `header-login-btn` visibility first, falling back to `mobile-menu-toggle` → `mobile-nav-login-btn` when it isn't visible.

**Root Cause 3 (self-inflicted, from CHG-861) — email exceeding RFC 5321's 64-char local-part limit.** CHG-861's own fix to `US-843-shipper-dashboard.spec.ts` generated per-test emails using `testInfo.testId` (a long Playwright-internal hash) for uniqueness, producing a 65-character local-part in the "adversarial" block (4 characters longer than the sibling "reskin" block, which stayed just under the limit). `TestAuthController`'s registration DTO has no validation and accepted it; the real `/api/v1/auth/login` endpoint's `LoginRequest` DTO has `@Email` Bean Validation and correctly rejected it with 400 — explaining why registration silently succeeded but login never navigated. Fixed by generating a short `Date.now()` + random-suffix email instead of embedding the full test ID.

**Files changed:** `frontend/e2e/design-system/US-844-carrier-load-board.spec.ts` (networkidle fix), `frontend/e2e/design-system/US-841-ui-primitives.spec.ts`, `US-842-layout-shell.spec.ts`, `US-843-shipper-dashboard.spec.ts`, `US-843-shipper-kpi-tiles.spec.ts`, `US-846-action-zone.spec.ts`, `US-854-diesel-region-caption.spec.ts`, `frontend/e2e/us-730a-v2-cost-profile-wizard.spec.ts` (mobile-menu fix), `US-843-shipper-dashboard.spec.ts` (email-length fix, same file as above).

**Verification (local Docker test env, single worker, isolated per-file runs):**
- `US-844-carrier-load-board.spec.ts`, `US-841-ui-primitives.spec.ts`, `US-846-action-zone.spec.ts`, `US-843-shipper-dashboard.spec.ts` run together: 26/31 passed (5 failures all traced to Root Cause 3 before that fix landed).
- `US-843-shipper-dashboard.spec.ts` alone after the email-length fix: **8/9 passed, 1 skipped, 21 seconds** (down from 90s-per-test timeouts).

**Status:** CHG-862 — all three root causes fixed and verified locally; PR pending, real CI run is the final confirmation.
