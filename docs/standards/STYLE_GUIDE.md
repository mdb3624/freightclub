# **AxleStack: Brand & Design System Reference**

This document defines the visual and verbal identity for **AxleStack**, a backend-first logistics engine. Use these tokens and principles for all frontend development and content generation.

## **1. Core Identity & Voice**
*   **Mission:** To provide the mechanical necessity of an axle, integrated with the agility of a software stack.
*   **Tagline:** The Load-Bearing Logic for Modern Freight.
*   **Brand Values:**
    *   **Structural:** Load-bearing infrastructure; unyielding.
    *   **Integrated:** Modern ecosystem; seamless API integration.
    *   **Unyielding:** Built to not fail; durable architecture.
*   **Voice & Tone:** Confident, Clear, Structural-focused, and Forward-looking.

### **Typography**
*   **Headlines / Display:** **SORA** (Sans-Serif).
*   **Body / UI / Code:** **INTER** (Sans-Serif).
*   **Style:** Clean, geometric, and high-legibility for technical data.

---

## **3. Component Guidelines**

### **Iconography**
*   **Style:** Use consistent, structural line icons with rounded corners.
*   **Library Preference:** Lucide-React or similar clean SVG sets.

### **Dashboard UI Pattern**
*   **Layout:** Dark mode interface with high-contrast data visualization.
*   **Metric Cards:** Use large typography for primary numbers (e.g., $2.48M Revenue) with small trend sparks.
*   **Charts:** Use Kinetic Blue and Accent Teal for line and donut charts to indicate activity and status.
