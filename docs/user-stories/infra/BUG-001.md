# BUG-001 — Write `claims` Row on Every Claim and Release

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want every load claim and release to insert a row into the `claims` table so that the authoritative claim audit trail exists for ratings, cancellation notifications, and bidding.

## Background

The `claims` table migration existed but `LoadService.claimLoad()` never inserted into it. This meant Phase 4 ratings (rater/ratee linked to a claim), Phase 2 cancellation notifications, and Phase 8 bidding all lacked a data foundation.

## Acceptance Criteria

- `LoadService.claimLoad()` inserts a `Claim` row with `status = ACTIVE` when a load is claimed
- `LoadService.cancelLoad()` updates the active `Claim` row to `status = CANCELLED` and sets `released_at`
- `LoadService.unclaimLoad()` (if applicable) updates to `status = RELEASED`
- Every claim transition is reflected in the `claims` table
- Unit tests cover claim insert and release update
