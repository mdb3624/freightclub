# TST-004 — Claim Load Concurrency Integration Test

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As a developer, I want a concurrency integration test for `claimLoad` so that the race condition fix (SEC-001) is proven to hold under simultaneous requests against a real database, not just in theory.

## Acceptance Criteria

- Test fires 2 simultaneous `claimLoad` requests for the same load using `ExecutorService`
- Exactly one request succeeds with the load in `CLAIMED` status
- The other request receives a 409 Conflict or equivalent failure
- The `loads` table has exactly one active claim (`trucker_id` is set to exactly one trucker)
- The `claims` table has exactly one `ACTIVE` claim row for the load
- Test uses `@SpringBootTest` with a real (H2 or test MySQL) database
