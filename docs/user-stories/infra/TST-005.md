# TST-005 — Frontend Claim Flow Smoke Tests

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As a developer, I want frontend smoke tests for the claim flow so that the most critical trucker action — browsing loads, claiming, and seeing confirmation — has automated coverage that catches regressions before they reach production.

## Acceptance Criteria

- Claim button renders on an open load card when the trucker has no active load
- Clicking claim fires the `claimLoad` mutation with the correct load ID
- A success toast appears after a successful claim response
- The load board disables claiming (or shows explanation) when the trucker already has an active load
- Tests use Vitest + React Testing Library with mocked API calls (no real backend needed)
