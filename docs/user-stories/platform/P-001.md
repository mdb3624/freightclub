# P-001 — Shared App Shell / Layout Component

**As** any user of the platform,
**I want** a consistent navigation and page layout shared across all pages,
**so that** the UI feels cohesive and I don't encounter jarring layout inconsistencies as I navigate.

## Acceptance Criteria

- [ ] A single `AppShell` (or equivalent) component wraps all authenticated pages and provides: top navigation bar, sidebar (if applicable), and page content area.
- [ ] The nav bar is identical across the shipper dashboard, load board, profile, and all other pages.
- [ ] Active page is indicated in the navigation (highlighted link or tab).
- [ ] No individual page component manages its own top-level layout (nav, background, max-width wrapper).
- [ ] The shell handles loading and error states for the current user (e.g., shows skeleton nav while user data loads).
- [ ] The component is responsive: collapses to a mobile-friendly layout on small screens.
- [ ] Extracting the shared shell causes no visual regression on any existing page.
