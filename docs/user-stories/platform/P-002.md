# P-002 — URL-Based Filter State for Load Board

**As** any user of the load board,
**I want** filter and sort state reflected in the URL as query parameters,
**so that** I can bookmark filtered views, share specific filter combinations, and use browser back/forward without losing my context.

## Acceptance Criteria

- [ ] All load board filters (origin state, destination state, equipment type, pickup date) are serialized as URL query params on change.
- [ ] Sort column and sort direction are also reflected in the URL.
- [ ] URL updates happen via `history.replaceState` (or React Router equivalent) — no page reload on filter change.
- [ ] Navigating back from a load detail page restores the full filter/sort state.
- [ ] Opening a URL with query params pre-applies those filters on load.
- [ ] Invalid or unrecognized params are silently ignored; the board defaults to showing all open loads.
- [ ] The implementation is shared between the trucker load board and any shipper-facing views that use the same filter pattern.
