# P-003 — In-App Notification Bell with Unread Count

**As a** user (shipper or trucker),
**I want** an in-app notification bell showing an unread count,
**so that** I can see at a glance whether something needs my attention.

## Acceptance Criteria

- [ ] A bell icon appears in the top navigation bar on all pages
- [ ] The bell displays a red badge with the number of unread notifications (e.g. "3")
- [ ] The badge is hidden when the count is zero
- [ ] Clicking the bell opens a dropdown or side panel showing a list of recent notifications
- [ ] Each notification in the list shows: type (e.g. "Load Claimed"), load ID, timestamp, and brief message
- [ ] Notifications are marked as read when the user clicks on them or views the full notification
- [ ] After marking a notification as read, it disappears from the unread count immediately
- [ ] The unread count is persisted in the database (not in React state) so it survives page refresh
- [ ] Notifications are sorted newest first
- [ ] The dropdown shows at least the 20 most recent notifications
