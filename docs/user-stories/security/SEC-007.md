# SEC-007 — CORS Explicit Header Whitelist

**Phase:** 1.2
**Persona:** Platform
**Status:** ✅ Built

## Story

As the platform, I want CORS `allowedHeaders` to use an explicit whitelist rather than `*` so that the API does not blindly accept arbitrary request headers from any origin.

## Acceptance Criteria

- `SecurityConfig` sets `allowedHeaders` to an explicit list: `Authorization`, `Content-Type`, `X-Requested-With`
- Requests with headers outside the whitelist are rejected with 403
- The existing frontend API calls (login, loads, auth refresh) continue to work without error
- `allowedHeaders: ["*"]` is not present anywhere in the config
