# S-007 — Address Field Order Correction

**As a** shipper,
**I want** address fields ordered street → city → state → zip,
**so that** the form matches the US convention I'm used to and reduces data entry errors.

## Acceptance Criteria

- [ ] All address input groups (origin and destination) render fields in order: Street, City, State, Zip.
- [ ] Tab order follows the same top-to-bottom sequence.
- [ ] Labels match exactly: "Street", "City", "State", "Zip" (or "ZIP Code").
- [ ] No functional behavior changes — only the visual and tab order is corrected.
- [ ] The corrected order applies on both the load creation form and the load edit form.
