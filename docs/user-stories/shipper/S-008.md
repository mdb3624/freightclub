# S-008 — Cross-Field Date Validation

**As a** shipper,
**I want** the system to validate that date windows are logically ordered,
**so that** I cannot post a load with impossible or nonsensical date windows.

## Acceptance Criteria

- [ ] `pickupTo` must be after `pickupFrom`. Violation shows: "Latest Pickup must be after Earliest Pickup."
- [ ] `deliveryFrom` must be after `pickupTo`. Violation shows: "Earliest Delivery must be after Latest Pickup."
- [ ] `deliveryTo` must be after `deliveryFrom`. Violation shows: "Latest Delivery must be after Earliest Delivery."
- [ ] Validation runs on form submit and also inline when the user leaves a date field (onBlur).
- [ ] The form cannot be submitted while any date validation error is active.
- [ ] Error messages appear adjacent to the offending field, not only in a top-level banner.
- [ ] All four date fields are required; blank date fields show a "Required" error on submit.
