# S-021 — Load Status Timeline

**As a** shipper,
**I want** a full status timeline per load,
**so that** I have an audit trail for disputes and billing.

## Acceptance Criteria

- [ ] Load detail page includes a "Timeline" or "Activity" section.
- [ ] Each status transition is recorded as a timestamped event: status, timestamp (UTC displayed in local time), and actor (shipper name or trucker name).
- [ ] Events include: DRAFT created, Published (DRAFT → OPEN), Claimed, Picked Up (IN_TRANSIT), Delivered, Cancelled (with reason if provided).
- [ ] Timeline is displayed in chronological order, oldest to newest.
- [ ] Timeline is read-only.
- [ ] Timeline is visible to both the shipper and the assigned trucker.
